# mdeberta-n2sf(base) 0.8 돌파 전략

> 일자: 2026-07-02 · 목표: 백본 교체 없이 **base 뉴럴-단독 macro-F1 0.72~0.78 → 0.8+**
> 결론: base가 막힌 건 백본 한계가 아니라 **레시피 문제**(S 데이터 기아 + 가짜 soft라벨 + 길이 불일치). 교정 시 0.8 도달 가능성 높음(보장 아님).

---

## 0. "0.78은 최종인가?" — 아니다

- 0.78 = 3차 테스트 base 뉴럴-단독 / 0.719 = 라인업 신선 재증류. **둘 다 현재 지점이지 상한 아님.**
- 아래 3대 병목이 미교정 상태의 수치 → 교정 후 재측정 필요.

## 2. 전략 (ROI 순)

### ★ 우선순위 1 — S 클래스 기아 해소 (최대 효과)
가장 큰 병목. 셋 중 조합:
- **(a) 클래스 균형 샘플링**: DataLoader에서 O/S/C 등확률 추출(S 오버샘플) — 코드 소량.
- **(b) S 경계 문서 대량 생성**: `gen_llm.py`로 S↔C, S↔O 경계 문서를 교사로 추가 생성·라벨 → S 비중 ↑.
- **(c) 클래스 가중 손실**: soft-CE에 클래스 역빈도 가중(S 가중↑).
> 기대: S 재현율 회복이 macro-F1을 가장 크게 올림.

### ★ 우선순위 2 — 진짜 soft-label 복원 (KD 이점 회수)
교사 라벨이 near one-hot이라 증류가 하드학습과 다름없음. 해소:
- **온도 스케일링(T>1)**: 목표분포를 `softmax(log p / T)`로 재-소프트화(T=2~3). 즉시 적용 가능(코드 몇 줄).
- **가능하면 교사 logprobs 수집**: argmax류 확률 대신 실제 로그확률/상위-k → 경계 뉘앙스 보존(재라벨 필요).
- **하드 CE 소량 혼합(α)**: `L = α·softCE(T) + (1−α)·CE(hard)` — 안정화.
> 기대: 경계(S) 판별 뉘앙스 전이 → S/O·S/C 오분류 감소.

### ★ 우선순위 3 — maxlen 정합 + 학습 스케줄
- **훈련 maxlen 256**으로 평가와 일치(문서 뒷부분 PII·키워드 보존).
- **LR warmup(10%) + cosine decay**, **4~5에폭**, **held-out 기준 early-stop/best 체크포인트 저장**.
- 유효배치 유지(batch16×accum2). MPS 시간예산 내(0.7→~1.2h).
> 기대: train/eval 분포 정합 + 안정 수렴.

### 보조 (여력 시)
- **체크포인트 평균(SWA)**: 마지막 2~3 체크포인트 가중평균 — 값싼 일반화 이득.
- **held-out 확대(72→150~200)**: 측정 노이즈 축소(현재 문서당 ±1.4%p).
- **하드 예제 마이닝**: 교사 엔트로피 높은/오답 문서 가중 재학습.

## 3. 실행 계획 (한 번의 재증류 run)

```
1) train_soft.py 개선:
   - 클래스 균형 샘플러(또는 가중 손실) 추가
   - KD 온도 T(기본 2.0) 옵션 + 하드 CE 혼합 α(기본 0.1)
   - maxlen 기본 256, LR warmup+cosine, best-on-heldout 저장
2) (선택) gen_llm.py로 S 경계 문서 +N건 생성·교사 라벨 → 교사셋 S비중 보정
3) models/n2sf-base 재증류 → held-out 재평가(metrics.py)
4) 0.8 미달 시: T·α·클래스가중·에폭 소폭 스윕(하네스)
```

**예상 소요**: 개선 코딩 + 재증류 1~2h(MPS) + 평가. 경계문서 생성 추가 시 +교사호출 시간.

**정직한 기대치**: base 279M의 현실적 상한은 대략 **0.80~0.85**(교사·용량 제약). 0.8 도달 가능성 높지만 보장 아님. 그 이상 확실히 원하면 **XLM-R-large(0.917)** 로 승격이 답.

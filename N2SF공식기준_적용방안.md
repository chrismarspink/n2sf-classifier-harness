# N²SF 공식 가이드라인 적용 방안 — 현행 모델과의 차이·전환 계획

> 기준: 국정원 「국가 망 보안체계(N²SF) 보안 가이드라인 1.0」 · 일자: 2026-07-07
> 결론: **적용 가능하나 "라벨 재정의"** (임계 튜닝 아님). 현행 모델은 공식 기준과 **개인정보 등급이 반대** → 재라벨·재증류 필수.

---

## 0. 판정 — 어느 기준이 옳은가 (기존 vs N²SF)

**N²SF 적용(개인정보=S)이 올바르다.** 이유:
- 본 제품은 **N²SF 등급분류기** → **국정원 공식 가이드라인이 정답의 정의**. 개인정보=§9-6호=**S**.
- 기존 "개인정보=C"는 우리가 만든 **프록시**(정규식으로 잡기 쉬운 강식별자를 최고등급化) — 이는 **DLP(개인정보 유출 심각도)** 관점이지 **N²SF 영향도 등급**이 아님. **N²SF로는 비준수(오분류)**.
- 두 프레임워크는 다른 질문에 답함: "국가 이익 영향?"=N²SF(개인정보 S) vs "보호 개인정보 존재?"=DLP(개인정보 심각).

**단서**: N²SF에서도 개인정보는 **S로 확실히 보호**(무시 아님). **기본 S, 맥락 상향**(수사·외교 자료·보안업무규정 비밀이면 C).
고객이 더 엄격히 원하면 **GradeProfile 상향 규칙**으로 override(표준=N²SF, 옵션=상향).

**함의**: 지금까지 0.9x는 "우리 라벨(부분 오류)에 맞나"를 잰 것 → N²SF 재라벨은 **성능 하락이 아니라 정답지 교정**.

---

## 1. 공식 기준 매핑 (정보공개법 §9 기반)

| 등급 | 정의(영향 기반) | 정보공개법 §9 | 대표 |
|---|---|---|---|
| **C 기밀** | 국가안보·외교·국방·수사·국민생명안전에 **중대 영향** | **제1~4호** + 보안업무규정 비밀 | 외교협상·군사작전·수사·정보기관 자료 |
| **S 민감** | 개인·기관 이익 침해 가능(비공개) — **공공정보 대부분** | **제5~8호**(내부검토·**개인정보**·영업비밀) | **개인정보(주민번호 포함)**·내부문서·인사·예산·로그·백업 |
| **O 공개** | C·S 아님(잔여) + 공개조치/기간경과 | 해당 없음 | 보도자료(발표후)·공개통계 |

---

## 2. ★핵심 불일치 — 반드시 교정 (현행 vs 공식)

| 항목 | 현행 우리 엔진 | 공식 가이드라인 | 조치 |
|---|---|---|---|
| **주민번호·개인정보** | **C**(KR_RRN 6.0, 단독→C) | **S**(§9-6호) | **S로 재정의** ← 최중요 |
| C의 본질 | **형태 기반**(강식별자 존재) | **영향 기반**(외교·국방·수사) | 의미 판단으로 전환 |
| 대량 개인정보(명부) | C | 개인정보이므로 **S**(단 위험도↑) | S 기본, 정책상 상향은 옵션 |
| 기밀표지(기밀/대외비/극비) | C 하드floor | 보안업무규정 비밀=C에 부합 | 유지(단 '대외비'는 법적 검토) |
| 로그·임시·백업 | (미분류) | **S 명시** | S 규칙 추가 |

> 현행 모델을 그대로 쓰면 **개인정보 문서를 C로 과대분류** → 공식 기준 위반. 전환 없이는 부적합.

---

## 3. 두 가지 본질적 난제 (형태→영향 전환의 대가)

1. **C가 의미·영향 기반이 됨** — "외교 협상 자료/수사 자료"는 **정규식·PII로 검출 불가**.
   → C 판정은 **LLM 티처 + 뉴럴의 의미 이해**에 의존. PII 기반 검출은 대부분 **S 판정기**로 재배치.
   → 규칙 tier의 강식별자는 이제 **S floor** 역할(C floor는 비밀표지·국가안보 맥락으로 축소).
2. **O vs S(및 상태 변화)는 메타데이터 필요** — "보도자료 초안=S → 발표후=O"는 **텍스트만으로 불가**.
   → **공개여부·시점 메타데이터** 결합 필수. innoECM 라벨토큰(상태 필드)과 연동. 순수 텍스트 분류의 상한.

---

## 4. 적용 절차 (전환 계획)

1. **라벨 rubric 교체**: `골드셋_라벨링_지침.md`를 §9 매핑으로 개정(C=1~4호, S=5~8호·개인정보, O=잔여+공개조치).
2. **티처 POLICY 프롬프트 재작성**(`harness/llm_baseline.py`): §9 각 호를 명시해 라벨 인버전 방지
   (예: "주민등록번호=§9-6=S, 외교·수사=§9-2·4=C").
3. **규칙 재매핑**(`data_classifier.py`):
   - KR_RRN·개인정보 → **S** 기여(현 C 임계에서 하향). 기밀표지 하드floor는 C 유지.
   - 로그/백업/임시 키워드 → S. 대량 개인정보 → S(정책 옵션으로 상향 가능).
4. **재라벨 → 재증류**: 개정 POLICY로 교사 soft-라벨 재생성 → xlmr 재증류.
5. **사람 골드셋도 §9 rubric으로** 라벨(이미 추출한 447건에 개정 지침 적용).
6. **양방향 오류 평가**: over-classification(공개→상위)도 규범 위반 → precision·recall 동시 보고.

---

## 5. 예상 영향 (정직)

- **수치 재정렬**: 개인정보=S로 옮기면 라벨 분포·점수가 크게 바뀜(현재 "C 잘 잡음"의 상당수가 실은 S). **낮아 보일 수 있으나 이게 올바른 기준.**
- **C는 어려워짐**(의미 기반) → C recall 확보가 관건. 보수적 상향(high-watermark)은 가이드라인 근거로 정당.
- **S가 중심 클래스**(공공정보 대부분) → S 데이터·경계(S↔O) 집중 필요.
- **O는 메타데이터 의존** → 텍스트 분류 + 상태정보 결합 아키텍처로.

---

## 6. 사용자 문서 §5 유의점 = 그대로 채택
S 중심·시점의존·O 잔여표현·보수적 상향·로그=S·과잉분류 방지 — 모두 정확. 구현에 반영.

---

## 7. 권고 (다음 스텝)
1. **1순위(즉시)**: 티처 POLICY + 규칙 매핑을 §9 기준으로 재작성(코드) → 소규모 재라벨·재증류로 **개념검증(PoC)**.
2. **2순위**: 메타데이터(공개여부·시점) 입력 경로 설계(O 판정 정확도의 상한 결정).
3. **3순위**: 사람 골드셋을 §9 rubric으로 라벨 → 공식 기준 대비 진짜 정확도·SLA.

> 요약: **적용 가능. 단 "개인정보=C"인 현행을 "개인정보=S"인 공식 기준으로 재정의**하는 게 핵심이며,
> C의 영향 기반 판단(외교·수사)과 O의 메타데이터 의존이 새 난제. 재라벨·재증류로 전환하는 게 정도(正道).

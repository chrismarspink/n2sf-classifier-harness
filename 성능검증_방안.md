# 성능 검증 방안 — "너무 좋다"는 의심을 어떻게 확인하나

> 일자: 2026-07-06 · 질문: base-ml 0.972, 다국어 0.99~1.0이 너무 높다. 검증 방안?
> 요지: **즉석 진단 3종은 "가짜"의 흔한 원인(누출·쉬운문제·패턴암기)을 반박**했으나,
> **같은 생성기 분포·표본 72·LLM 라벨**이라는 근본 의심은 남아 있음 → 아래 검증으로 확정 필요.

---

## 1. 이미 확인한 것 (즉석 진단 결과)

| 진단 | 결과 | 의미 |
|---|---|---|
| **train/test 유출·중복** | 정확중복 0/72, 근접유사(4-gram Jaccard) median 0.01·max 0.10 | 학습셋 복사·오염 **아님** |
| **규칙만(뉴럴 OFF)** | F1 **0.558** (과대분류 0.36) | 규칙만으론 못 풂 → **쉬운 세트 아님**, 뉴럴이 0.558→0.972로 실제 기여 |
| **PII 마스킹 후** | 0.972 → **0.778** (규칙 0.558보다 높음) | 문자열 패턴만이 아니라 **의미도 학습**(암기 아님) |

→ "누출/쉬운문제/패턴암기"라는 흔한 허위원인은 배제. 그럼에도 아래가 남는다.

---

## 2. 남아 있는 진짜 의심 (미해소)

1. **동일 생성기 분포**: held-out을 학습 라벨과 **같은 LLM 계열(gpt-4o/o4-mini)** 로 생성 → n-gram 중복은 없어도
   **문체·구조 분포가 유사**할 수 있음(가장 큰 의심). 모델이 "N²SF 등급"이 아니라 "이 생성기 스타일"에 최적화됐을 가능성.
2. **표본 72**: 0.972 = 70/72. 표본이 작아 **신뢰구간이 넓음**(대략 ±5%p). 점추정 과신 위험.
3. **라벨이 LLM/정책 기반**(사람 아님): 교사·평가 라벨이 같은 정책에서 나오면 **순환**. 진짜 정답 대비 검증 안 됨.
4. **다국어 L5 0.99~1.0**: L5 평가셋과 다국어 학습셋이 **유사 합성 패턴**이라 in-distribution 성격.

---

## 3. 검증 방안 (우선순위)

### ★ 1순위 — 진짜 OOD 세트 (다른 생성기·실문서)
학습에 **안 쓴 출처**로 평가셋을 만들어 분포 의존을 깬다.
- **다른 LLM 생성기**: Claude 또는 로컬 Gemma로 문서 생성(학습은 gpt-4o 생성) → 문체 분포 이동.
- **실문서 계열**: 실제 공문서/HWPX 템플릿·뉴스·공개 보고서 기반(합성이라도 실제 서식).
- **라벨 신뢰**: 등급 결정요소(PII·키워드)는 우리 synth로 **주입**해 라벨을 정책으로 고정(생성기와 무관).
- 기대: 여기서도 0.9대면 진짜, 크게 떨어지면 분포 과적합.

### ★ 2순위 — 사람 라벨 골드셋 (상용화 필수)
- **200~500건을 사람이 직접 등급 라벨** → LLM/정책과 독립된 절대 기준.
- 교사(o4-mini)와 학생 정확도를 이 골드셋에서 재측정. **SLA(누락률) 근거**가 됨.
- 두 사람 교차검수로 라벨 일치도(κ) 확인 — κ 낮으면 등급 기준 자체가 모호(프롬프트/정책 손봐야).

### ★ 3순위 — 표본 확대 + 신뢰구간
- held-out 72 → **300~500**로 확대. **부트스트랩 신뢰구간** 보고(점추정 대신 [하한, 상한]).
- 등급·언어·포맷별 층화(각 셀 충분히).

### 4순위 — 강건성(robustness) 프로브
- **역번역/패러프레이즈**: 다른 LLM으로 held-out을 바꿔 쓴 뒤 재평가(문체 불변성).
- **신규 어휘·엔티티**: 학습에 없던 회사명·표현으로 경계 문서 생성.
- **적대셋 확대**: 완곡·띄어쓰기·다국어 혼재 등 L4/L5 난이도 상향.

### 5순위 — 전향적(prospective) 검증
- 모델 동결 후 **새로 수집한 문서**로 평가(시간적 분리 → 가장 현실적).

---

## 4. 지금 바로 할 수 있는 것 (제안)

1. **OOD held-out 생성·평가** — Gemma(로컬, 학습 미사용) + 다른 프롬프트 스타일로 문서 생성 + synth PII 주입 →
   전 모델 재평가. (반나절, 로컬/무API 가능) ← **가장 값싸고 강력한 반증 테스트**
2. **표본 확대** — 동일 파이프라인으로 held-out 300건 생성 후 부트스트랩 CI.
3. **PII-독립 하위세트** — PII 없이 문맥만으로 등급이 갈리는 문서만 모아 평가(의미 이해 순수 측정).

> 사람 골드셋(1·2순위 핵심)은 사람 손이 필요하니, 착수하시면 라벨링 지침·200건 후보 추출까지 준비하겠습니다.

---

## 5. 정직한 현재 판단

- 0.972가 **명백한 허위(누출·암기)는 아님**(진단 3종 통과). 뉴럴이 규칙 대비 +0.41 실제 기여.
- 그러나 **"같은 분포에서 0.97"** 이므로, **다른 분포(OOD)·사람 라벨**에서 재현되기 전엔 **상용 수치로 확정 금지**.
- 실무 권고: 대외 보고엔 **"동일 분포 held-out 0.97, OOD·실문서 검증 예정"** 으로 표기하고, 안전지표(누락 0)는
  다층 방어(규칙 floor)로 별도 보증.

# N²SF 등급분류 — 상업적 사용 가능성 의견

> 일자: 2026-07-03 · 전제: **온프레미스 + No GPU + 외부 LLM 미사용**(추론 시). 데이터 반출 0.
> 결론(요지): **규제 시장(공공·금융·국방·의료)에서 상업성 높음.** 특히 **ECM 내장 자동 등급분류 애드온**으로
> 강한 적합성. LLM급 정확도(0.9+)·설명가능·제로 반출·GPU 불필요가 핵심 차별점.

---

## 1. 시장 니즈 (왜 팔리나)

- **클라우드 금지 시장이 실재**: 국내 공공·금융·국방·의료는 **망분리·에어갭·데이터 국외반출 금지**로
  클라우드 LLM/DLP를 **못 쓰는** 경우가 많음 → 온프레미스 대안 수요가 구조적으로 존재.
- **N²SF(국가 보안 프레임워크) 준수 압박**: 문서 보안등급 분류는 규제 대응 필수 업무인데 **수작업은 비현실적**.
- **개인정보보호법·컴플라이언스**: 판단 근거 제시(감사) 의무 → 설명가능한 분류기 필요.

→ "클라우드는 못 쓰는데, 규정상 자동 분류는 해야 하는" **미충족 수요**를 정면으로 겨냥.

---

## 2. 차별점 (경쟁 우위)

| 축 | N²SF | 클라우드 LLM/DLP | 전통 규칙기반 DLP |
|---|---|---|---|
| 데이터 반출 | **0(온프레미스)** | 있음(반출) | 0 |
| 정확도 | **0.9+(LLM 증류)** | 0.9~0.97 | 낮음(의미 미이해) |
| GPU | **불필요(CPU)** | N/A | 불필요 |
| 추론 비용 | **0(자사 자원)** | 호출당 과금 | 0 |
| 설명가능성 | **SHAP·규칙 근거** | 블랙박스 | 규칙만 |
| 기밀 누락 | **0 지향(C재현율 1.0)** | 모델 의존 | 높음 |
| 고객 정책 | **GradeProfile 주입** | 제한적 | 수동 |

- **핵심 메시지**: "**외부 LLM·GPU 없이, 데이터가 회사를 벗어나지 않으면서, LLM급 정확도.**"
- 경쟁 클라우드 서비스는 **에어갭 요건에서 애초에 실격** → 우리는 그 시장을 사실상 독점적으로 접근.

---

## 3. 경제성 (비용 구조)

- **교사(LLM) 비용 = 개발 단계 1회성**(증류 학습). 추론엔 LLM 없음 → **추론당 과금 0**.
- **추론 = 고객 기존 CPU** → 한계비용 ~0, 추가 하드웨어(GPU) capex 불필요 → **낮은 TCO**.
- 마진 구조 우수: 소프트웨어 라이선스 + 유지보수(모델·정책 업데이트) 중심.

**라이선스 모델(안)**: 서버당/어플라이언스당 라이선스 + 연간 유지보수(모델 갱신·정책 튜닝) +
GradeProfile 커스터마이징 전문서비스(PS).

---

## 4. 우리에게 특히 유리한 각도 — ECM 내장

- 이노티움은 **ECM(문서 콘텐츠 관리)** 사업 기반 → N²SF는 **ECM에 자연스럽게 내장되는 애드온**.
- 시나리오: ECM 저장소에 문서가 들어올 때 **자동 등급분류→라벨·접근제어·감사로그** 연동.
- 기존 ECM 고객 베이스에 **업셀**(추가 모듈 판매) + 신규 고객엔 "보안 자동분류 내장 ECM"으로 차별화.
- Docker/온프레미스 납품 방식이 기존 ECM 배포와 동일 → **통합·영업 마찰 낮음**.

> 이게 가장 현실적인 1차 상업화 경로로 보임(신규 독립제품보다 리스크·비용↓, 시장진입 빠름).

---

## 5. 타깃 세그먼트 (우선순위)

1. **공공기관·국방** — N²SF 준수 의무, 망분리 강제 → 최우선.
2. **금융** — 개인정보·내부문서 등급관리, 규제 강함.
3. **의료·연구** — 민감정보(진료·연구자료) 다수.
4. **대기업 보안조직** — 사내 DLP·문서보안 고도화 수요.

---

## 6. 리스크 & 한계 (정직하게)

- **정확도 상한**: 온디바이스 라지 0.9+ (o4-mini 0.97엔 못 미침). → **안전(C재현율 1.0)으로 상쇄**하고,
  "누락 0 + 과대분류는 사람이 재검토" 운영으로 포지셔닝.
- **오분류 책임/SLA**: 상용화 시 **C재현율(누락률) SLA** 명문화 필요. 과대분류율·재검토 프로세스 정의.
- **모델 유지보수**: 정책·언어·문서유형 변화 시 재증류 필요 → 유지보수 계약으로 수익화 가능.
- **고객 데이터 재보정**: 도입처 실문서로 임계·정책 재보정해야 최적 → PS 기회이자 초기 공수.
- **개발단계 교사 거버넌스**: 증류에 **합성 문서만** 사용(실기밀 금지) 원칙·키 관리 프로세스 필수(현재 준수 중).
- **성능 편차**: 문서 유형·언어에 따라 편차 → 도입 전 파일럿으로 실측 제시 권장.

## 8. 권고 (의견)

1. **상업성 있음** — 특히 **에어갭 규제 시장 + ECM 내장**의 결합은 경쟁이 거의 없는 틈새. 진입 타당.
2. **1차 경로 = ECM 애드온**(기존 고객 업셀). 독립 제품화는 그다음.
3. **차별화 카피** = "데이터가 회사를 떠나지 않는 LLM급 자동 보안등급 분류(GPU 불요)".
4. **다음 상업화 스텝**:
   - 실고객 문서로 **파일럿·정확도 실측**(합성→실데이터 갭 확인).
   - **C재현율 SLA·재검토 워크플로** 정의.
   - **GradeProfile 전면 구현**(회사별 정책 세일즈 포인트).
   - N²SF/보안 인증 정렬, 레퍼런스 확보.
5. **가격**: 서버 라이선스 + 유지보수(모델갱신) + PS(커스터마이징) 3중 수익.

> 총평: 기술·안전·비용 구조가 **규제 온프레미스 시장에 정확히 들어맞음**. ECM 결합 시 즉시 상업화 가능한
> 완성도(v1.1). 남은 건 제품화(정책 구현)·검증(파일럿/SLA)·인증이며, 기술 리스크보다 **영업·검증 리스크**가 주.

## 0. 목표 (요청 요지)

`data_classifier.py` 는 정규식 + Presidio NER + 경량 BERT 3-tier 로 문서를 **C(기밀)/S(민감)/O(공개)**
로 분류하는 모델. **외부 LLM·GPU 없이 LLM 수준 성능**을 내는 것이 목표.
이를 위해:
1. 등급·포맷별 테스트 데이터 생성 → 분류 → KPI 측정
2. 정규식·NER·키워드·뉴럴·앙상블·임계값을 바꿔가며 **최적 설정 자동 탐색**
3. 결과를 DB/Excel/웹으로 정리, 성능(속도)도 측정
4. 비교군 LLM(처음 Claude, 이후 로컬 Gemma2:9b)과 정량 비교
5. 주말 동안 무인으로 **난이도를 올려가며 반복 최적화**

수천 조합 스윕 → 조합 폭발을 CPU에서 감당. (점수 단계 지연 0.01ms 실측)

## 2. 만든 것 — 하네스 (`harness/`)

```
generate → detect(캐시) → optimize(2계층 탐색) → LLM 비교 → report(Excel/HTML/설정 산출)
```

| 모듈 | 역할 |
|---|---|
| `corpus.py` | 등급×난이도(L1~L4)×9포맷 합성 테스트셋. **N²SF 정책 기준 라벨**(분류기 규칙과 분리=순환오류 방지). PII 체크섬·Luhn 유효 |
| `detect.py` | 추출+Presidio NER+뉴럴 1회 실행·캐시(정규화 해시로 포맷 변이 중복제거) |
| `score.py` | 캐시 위에서 점수·앙상블 재생(분류기 실제 함수 재사용→결과 일치) |
| `metrics.py` | 혼동행렬·F1·과소/과대분류율·지연 + **안전 우선 목적함수**(C재현율 미달 강감점) |
| `optimize.py` | 2계층 탐색: 규칙+NER 점수설정 → 뉴럴×앙상블 → tier 가중치 |
| `llm_baseline.py` / `ollama_baseline.py` | 비교군 LLM (Claude API / 로컬 ollama gemma2:9b) |
| `summarize.py` | 난이도별·모델별 집계 + 추천 최종 설정 |
| `report.py` / `visualize.py` | Excel 리포트 / 발표용 HTML 대시보드(인라인 SVG, 외부 의존 0) |
| `autoloop.py` | 난이도 에스컬레이션 무인 루프(주말 실행) |

**포맷 9종**: txt, csv, json, md, docx, **xlsx**, pptx, **hwpx**, pdf (동일 내용을 전 포맷 렌더 → 추출 레이어 결함 분리). xlsx/hwpx 는 표준 라이브러리로 분류기가 읽는 XML 직접 생성.

## 3. 핵심 결과

### 3.1 단일 실행 최적화 (test)

| 설정 | macro-F1 | 정확도 | 기밀 재현율 | 과대분류 | 속도(CPU) |
|---|---|---|---|---|---|
| 기본(default) | 0.47 | 0.56 | 1.00 | **0.44** | ~8ms |
| **규칙+NER 최적** | 0.94 | 0.94 | 1.00 | 0.06 | **~8ms** |
| **규칙+NER+mdeberta/vote** | **1.00** | **1.00** | 1.00 | **0.00** | ~812ms |

**하네스가 자동 발견한 결함 2건**:
- 전화번호가 느슨한 계좌 정규식(`\d{2,6}-\d{2,6}-\d{2,7}`)에 오탐 → S를 C로 대량 과대분류(기본 44%). 가중치·임계값으로 교정.
- "기밀이 포함되지 않음" 부정문맥 키워드 오탐 → 규칙 불가, mdeberta(zeroshot)+vote 가 의미적으로 교정.

### 3.2 로컬 LLM(Gemma2:9b) 비교 (test 162파일)

| 모델 | macro-F1 | 기밀 재현율 | 지연 p50 |
|---|---|---|---|
| **최적 BERT 3-tier** | **1.00** | 1.00 | ~812ms |
| Gemma2:9b | 0.89 | 1.00 | ~18,900ms |

→ 쉬운 난이도에서 **3-tier 가 9B 로컬 LLM보다 정확·~23배 빠름(GPU 없이)**.
(Gemma 대표 오답: "기밀 아님" 부정문맥을 S로 과대분류 — 9B도 함정에 빠짐)

### 3.3 주말 무인 최적화 (84회 반복, L1→L3)

| 난이도 | 내용 | 최적 모델 | macro-F1 | 비고 |
|---|---|---|---|---|
| L1 깨끗 | 정상 문서 | mdeberta/weighted | **0.99** | 거의 완벽 |
| L2 경계 | 연락처 4~9명(bulk 임계) | mdeberta/soft | **1.00** | 해결 |
| L3 적대 | `대 외 비` 공백·전각, 가짜 주민번호, 미끼카드 | (정체) | **~0.54** | **무너짐** |

- L1·L2 는 3-tier 로 LLM 수준 달성. **L3(적대 난독화)에서 정체**(72회 반복해도 돌파 못 함, 가끔 기밀 누락).
- L4 미도달(L3에서 막혀 자원 집중 — 설계대로).

## 4. 최적 BERT 모델: **mdeberta**

- 정체: **mDeBERTa-v3-base** (Microsoft, disentangled attention) 의 다국어판을 MNLI+XNLI 로 파인튜닝한
  **제로샷 NLI 분류기** (`MoritzLaurer/mDeBERTa-v3-base-mnli-xnli`, MIT).
- 동작: 문서를 전제로, 등급별 가설("이 문서는 기밀이다" 등)의 함의 확률 → 최고 등급 채택 → 앙상블 결합. **학습 불필요(zero-shot)**.
- 사양(실측): 파라미터 **279M**, 디스크 552MB, 메모리 ~1.1GB(int8 ~0.3GB), CPU ~0.5~0.8s/문서.
- **전 난이도 1위**(L1 0.94/L2 0.95/L3 0.84), 유일하게 1.00 달성.
- ⚠️ "kcbert 67승"은 착시 — L3에서 max-rank 앙상블이 "애매하면 무조건 상향"으로 안전점수만 가져간 것(실제 F1 0.55). 진짜 분별력은 mdeberta가 최고.
- **로컬 디바이스 적합** ✅ (CPU·일반 노트북 가능, 양자화로 더 경량).

## 5. 속도 분석 — early-exit 가정

운영 시 **Tier1 정규식이 강식별자를 잡으면 즉시 확정, 이후 단계 생략** 가정:

| 경로 | 지연/문서 | Gemma 대비 |
|---|---|---|
| Tier1 정규식 확정 (테스트의 **72%**) | **3.1 ms** | ~6,000배 |
| 뉴럴까지 가는 경로 (나머지 28%) | ~1,517 ms | 12배 |
| **early-exit 평균** | **≈ 424 ms** | **45배** |
| Gemma2:9b | 18,937 ms | 1배 |

→ 강식별자·기밀라벨이 명시된 실문서에선 early-exit 비율이 더 높아 **수십 배 우위**가 견고.

## 7. L3(적대 난독화) 돌파 — 권장 3단계

1. **전처리 정규화**: 키워드 매칭 전 공백 제거·전각→반각 변환 → `대 외 비`·전각숫자 무력화 (빠르고 확실).
2. **체크섬 검증**: 주민번호 검증자리·카드 Luhn 확인 → 가짜 식별자 오탐 제거(정밀도↑).
3. **mdeberta 파인튜닝**: 제로샷 대신 우리 라벨 데이터(특히 L3 1.7만건)로 분류 헤드 학습 →
   위장 패턴을 예시로 일반화. **오프라인 1회 학습, 추론은 여전히 CPU 소형 모델**(제약 위배 아님).
   - 데이터 이미 확보(하네스 생성), Apple Silicon 수십분~수시간, 과적합 방지 위해 실문서 표본 혼합 권장.

## 14. 환경·설치

- OS: macOS (Apple Silicon, 맥미니) · Python **3.12.13** (venv `.venv/`)
- 핵심: `presidio-analyzer`, `spacy` + `ko_core_news_sm`, `numpy`, `openpyxl`, `pyyaml`, `reportlab`, `pypdf`
- 뉴럴: `torch`, `transformers`, `sentence-transformers`, `sentencepiece`
- LLM 비교: `anthropic`(Claude, 키 필요) 또는 로컬 `ollama` + `gemma2:9b`(키 불필요)
- BERT 백엔드 12종 평가: minilm, ko-sroberta, klue-roberta, koelectra, kcbert, mbert, xlm-roberta, mpnet, e5, **mdeberta**, xlmr-xnli, labse

```bash
python3.12 -m venv .venv && . .venv/bin/activate
pip install presidio-analyzer spacy numpy openpyxl pyyaml reportlab pypdf \
            torch transformers sentence-transformers anthropic sentencepiece
python -m spacy download ko_core_news_sm
```

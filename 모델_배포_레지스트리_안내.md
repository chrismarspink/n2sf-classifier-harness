# 학습된 모델을 어디에 올리나 — 공개 레지스트리 안내

> 일자: 2026-07-02 · 질문: "모델 학습이 완료되면 어디에 올리나? 도커허브처럼 공개로 받을 수 있는 일반 레지스트리가 있나?"
> 답: **있다 — Hugging Face Hub가 사실상 표준("모델계의 Docker Hub")**. 용도별로 아래에서 선택.

## 1. 한눈에

| 목적 | 권장 | 비고 |
|---|---|---|
| **모델 가중치 공개 공유** | **Hugging Face Hub** | 사실상 표준. 대용량 자동 처리(Git LFS) |
| **분류기 "서비스"째 배포** | **Docker Hub / OCI 레지스트리** | 모델+서버+코드를 이미지로 패키징 |
| **사내·고객 납품(온프레미스)** | 사설 HF repo / 사내 아티팩트(Nexus·Artifactory·S3/MinIO) / 파일 번들 | 데이터주권·에어갭 |
| **클라우드 MLOps** | AWS SageMaker · Azure ML · Vertex AI Model Registry | 각 클라우드 종속 |

## 2. Hugging Face Hub = 모델계의 Docker Hub (핵심)

- **무엇**: 전 세계 표준 모델 저장소. 공개/비공개 repo 모두 지원. 무료 공개 호스팅.
- **왜 GitHub이 아니라 HF인가**: GitHub은 파일 100MB 제한이라 GB급 가중치(safetensors)를 못 올림
  (그래서 우리 GitHub엔 코드·문서만 올리고 모델은 제외했음). **HF Hub은 Git LFS로 대용량을 기본 처리.**
- **올리기**:
  ```bash
  pip install huggingface_hub
  huggingface-cli login                       # 토큰 입력
  huggingface-cli upload innotium/n2sf-xlmr-large ./models/n2sf-xlmr-large
  # 또는 코드로: model.push_to_hub("innotium/n2sf-xlmr-large")
  ```
- **받기(누구나)**: `AutoModelForSequenceClassification.from_pretrained("innotium/n2sf-xlmr-large")`
  → 우리 `data_classifier.py`의 `NEURAL_BACKENDS` model 경로를 이 HF ID로 바꾸면 자동 다운로드.
- **공개 범위**: Public(누구나) / Private(초대·토큰) / Gated(약관 동의 후) 선택 가능.

## 3. Docker Hub는? — "모델"이 아니라 "서비스" 단위

- Docker Hub는 **컨테이너 이미지** 레지스트리(모델 가중치 전용 아님).
- 우리 분류기는 이렇게 쓰면 딱 맞음: **모델 + `data_classifier.py` + API 서버**를 한 이미지로 묶어 배포.
  ```
  docker build -t innotium/n2sf-classifier:1.0 .    # 모델·코드 포함
  docker push innotium/n2sf-classifier:1.0
  # 고객: docker run -p 8080:8080 innotium/n2sf-classifier:1.0  → 즉시 분류 API
  ```
- 즉 **가중치만 공유 → HF Hub**, **바로 돌아가는 분류 서비스 배포 → Docker(이미지)**. 둘은 상호보완.
- (참고) OCI 아티팩트(ORAS)로 가중치를 OCI 레지스트리에 올릴 수도 있으나 일반적이진 않음.

## 4. 그 외 공개 레지스트리

- **ModelScope**(알리바바) — 중국권에서 HF 대안.
- **Kaggle Models**, **PyTorch Hub / TensorFlow Hub**(GitHub 기반) — 배포 채널로 사용 가능.

## 5. 온프레미스·상업 납품 (우리 제약에 맞는 선택)

데이터주권·에어갭이 중요한 우리 특성상, **공개 배포보다 통제된 배포**가 실무 핵심:

- **사설 HF repo**: 조직 전용, 토큰으로만 접근. 사용법은 공개와 동일.
- **사내 아티팩트 저장소**: Nexus·JFrog Artifactory(범용/LFS 지원), S3·MinIO 버킷 → 사내 표준 배포.
- **에어갭 고객**: 모델 폴더를 **tar 번들**로 오프라인 전달(USB·사내망). `from_pretrained("로컬경로")`로 로드.
- **Docker 이미지**(사설 레지스트리): 모델+엔진 통째로 → 고객이 `docker run` 한 줄로 구동.

> 상세한 물리적 실체·라이선스·버전관리는 `모델_실체_및_릴리즈_가이드.md` 참조.

## 6. 우리 프로젝트 권고 (역할 분담)

| 자산 | 어디에 | 이유 |
|---|---|---|
| **코드·문서** | GitHub (이미 반영) | 경량, 협업·이력 |
| **모델 가중치(공개용)** | Hugging Face Hub | 대용량 표준, `from_pretrained` 즉시 사용 |
| **모델 가중치(고객·사내)** | 사설 HF / Nexus·Artifactory / tar 번들 | 통제·에어갭 |
| **분류 서비스(즉시 구동)** | Docker 이미지(사설/공개 레지스트리) | 모델+엔진+API 일괄 배포 |

> 요약: **공개로 "다운받을 수 있게" = Hugging Face Hub**. 우리처럼 온프레미스 제품이면
> **사설 HF/사내 저장소 + Docker 이미지** 조합이 실무 정답. GitHub엔 코드만(가중치는 HF/사내로).

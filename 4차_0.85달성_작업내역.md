# 4차 테스트 — 과대분류 억제로 held-out 0.85 달성 작업내역

> 일자: 2026-07-02 · 출발점: 3차 최종 held-out F1 **0.797** (mdeberta-n2sf 증류 + 뉴럴우선 앙상블)
> 목표: **held-out macro-F1 ≥ 0.85** (기밀 재현율 1.00 = 누락 0 유지 필수)
> 제약: 외부 LLM·GPU 미사용, 온디바이스(CPU)·온프레미스 유지

---

## 1. 문제 정의

- 3차 최종 0.797의 잔여 손실은 **과대분류(over_rate 0.194)** — 정상/민감 문서를 상위 등급으로 오탐.
- 원인(규칙 tier FP):
  1. **전화번호 ↔ 계좌 정규식 오탐** — `010-xxxx-xxxx`가 느슨한 계좌 패턴(`\d{2,6}-\d{2,6}-\d{2,7}`)에 매칭되어 C 드라이버(6.0)로 오인.
  2. **키워드 substring 오탐** — "기밀이 아니다" 부정문맥의 '기밀'이 KEYWORD_SECRET로 잡힘.
  3. 강식별자 외 약한 신호들의 누적 escalation.
- **기밀 재현율(C)은 1.00 유지가 절대조건** — 과대분류를 줄이되 누락은 0이어야 함.

---

## 2. 작업 내역 (레버)

| # | 레버 | 방법 | 리스크 |
|---|---|---|---|
| 1 | **전화↔계좌 supersede** | 전화 span과 겹치는 계좌 finding 제거(`{"KR_ACCOUNT":["KR_PHONE"]}`) | 실계좌 누락? → 강식별자 다수는 유지 |
| 2 | **뉴럴 우선 앙상블** | soft + tier 가중 `neural≥3` → 증류 학생(0.78)이 규칙 FP를 하향 교정 | 뉴럴 오판 시 누락 → C재현율로 감시 |
| 3 | **임계 상향** | C≥8, S≥2 로 약신호 누적 escalation 억제 | 경계 민감도↓ → F1로 감시 |
| 4 | **KR_ACCOUNT 가중↓** | 6.0 → 1.0 (오탐 영향 축소, 실계좌는 문맥·뉴럴로 보완) | 실계좌 단독 문서 민감도↓ |
| 5 | **bulk 임계 조정** | 대량 PII escalation 임계 재검토(안전 우선 유지) | — |

**핵심 평가 원칙**: held-out(학습 미사용 LLM 분포)에서 **C재현율=1.00 제약 하 F1 최대화**.
supersede는 공개 `weights` API 미지원 → 하네스 `score.predict`(캐시된 findings)로 스윕.

---

## 3. 실행 방법

1. held-out 72문서에 대해 탐지 캐시 생성(presidio + mdeberta-n2sf 뉴럴) — 1회.
2. `score.predict`로 설정(supersede×임계×tier×앙상블×엔티티) 스윕 → 각 metrics.
3. **C재현율=1.00 & under=0** 만족 중 **F1 최대** 설정 선택.
4. ≥0.85 달성 시 → 산출물(모델·참조코드·백서) 생성. 미달 시 → 최적치 보고 + 다음 레버(교사 o4-mini 증류/더 큰 인코더).

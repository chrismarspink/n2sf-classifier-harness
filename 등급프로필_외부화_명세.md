# 등급 프로필 외부화 — Claude Code 빌드 명세

> **대상 파일:** `data_classifier.py` (단일 소스)
> **목표:** 모델은 **등급 불가지론(grade-agnostic) 범용 엔진**으로 두고,
> 등급 체계(라벨·임계값·강제상향·규제)는 **프로필(GradeProfile)** 로 외부화한다.
> N²SF는 내장 기본 프로필이자 **1급 검증 대상**으로 유지한다.
>
> **상태:** 설계 명세(보존). 이 리팩터는 위험도가 있어 **유인(attended) 작업**으로 별도 진행 권장.
> 2차 테스트의 자동 무인 작업은 **mdeberta-n2sf 개발 + L3 돌파**에 집중한다(본 명세와 분리).

---

## 0. 전략 요약 (이 명세가 구현하는 결정)

- **모델은 범용으로 학습·배포한다.** 신경망 tier는 등급 개수를 모른다. 등급은
  런타임에 프로필이 주입하는 **가설 문장(NLI/zeroshot)** 으로만 들어온다.
- **N²SF는 1급 시민으로 검증한다.** N²SF 프로필에 한해 전용 평가셋 + **C-recall
  하드 게이트**를 둔다. 다른 업체 프로필은 경량 스모크 평가만.
- **결정론적 floor는 프로필별로 정의되며 침범 불가다.** 확률 tier(NER·신경망)는
  **상향만** 가능, 하향 불가.
- **에어갭(C-grade) 호환.** 모델·프로필 모두 로컬 경로에서 로드. 런타임 허브 접근 금지.

### 전제: 신경망은 zeroshot(NLI 가설) 형태

범용성이 "거의 공짜"가 되는 것은 모델이 **3-class 분류 헤드**가 아니라
**NLI 가설 형태**로 동작할 때다. 현재 `NEURAL_BACKENDS`의 `mdeberta`가 이미
`kind: "zeroshot"` 이므로 본 명세는 이를 범용 엔진의 기본 형태로 전제한다.

- `exemplar` / `embed` kind도 등급별 **예시문**을 프로필에서 받으므로 동일하게 범용.
- 특정 업체가 정확도 게이트를 못 넘는 경우에만, 그 업체 한정으로 3-class 헤드를
  파인튜닝한 **전용 변형 + 잠긴(locked) 프로필**을 예외로 추가한다(§8).

## 1. 현재 상태 — 등급 체계가 코드에 박혀 있는 지점

| 심볼 | 현재 내용 | 프로필 이전 후 |
|---|---|---|
| `GRADE_RANK` / `RANK_GRADE` | C/S/O 3단 고정 | 프로필의 **순서 있는 등급 리스트**에서 파생 |
| `SHORT` | `{OPEN:"O",...}` 코드 맵 | `profile.codes` |
| `GRADE_LABEL_KO` | 한국어 표시명 | `profile.labels` |
| `_GRADES` | `["OPEN","SENSITIVE","CONFIDENTIAL"]` | `profile.grades` |
| `C_THRESHOLD` / `S_THRESHOLD` | 5.5 / 0.75 | `profile.thresholds` (순서 있는 맵) |
| `ZS_LABELS` | locale→등급→가설문 | `profile.zs_hypotheses` |
| `EXEMPLARS` | locale→등급→예시문 | `profile.exemplars` |
| `ENTITY_WEIGHTS` | 타입별 점수 | base + `profile.entity_weights`(override) |
| `GRADE_KEYWORDS` | 등급 키워드 | base + `profile.keywords` |
| `BULK_PII_TYPES` / `BULK_PII_THRESHOLD` | 대량 PII 규칙 | `profile.floor_rules` |
| 마이넘버 강제 C (`classify_text` 내부) | 하드코딩 분기 | `profile.floor_rules` |
| `LOCALE_REGS` | locale→규제 | `profile.regulations` |

## 2. GradeProfile 스키마

```python
@dataclass(frozen=True)
class GradeProfile:
    name: str
    grades: list[str]                          # 낮음→높음 순서. rank = 인덱스
    codes: dict[str, str]
    full_names: dict[str, str]
    labels: dict[str, str]
    thresholds: dict[str, float]
    entity_weights: dict[str, float]
    keywords: list[tuple]
    zs_hypotheses: dict[str, dict[str, str]]
    exemplars: dict[str, dict[str, list[str]]]
    bulk_pii_types: set[str]
    bulk_pii_threshold: int
    floor_rules: list[dict]
    regulations: dict[str, list[str]]
```

### 핵심 일반화 포인트
- 등급은 순서 있는 리스트. rank는 인덱스. 3개 가정 제거.
- 임계값은 순서 있는 맵. `_score`는 최고 등급부터 내려오며 첫 충족 등급을 잡는다.
- 신경망은 `grades`/`zs_hypotheses`만 본다.

## 3. 내장 프로필 — N²SF (back-compat 잠금)

`profile="n2sf"` 일 때 **기존 동작과 바이트 단위 동일**해야 한다(§7 회귀 게이트).

```python
N2SF_PROFILE = GradeProfile(
    name="n2sf",
    grades=["OPEN", "SENSITIVE", "CONFIDENTIAL"],
    codes={"OPEN": "O", "SENSITIVE": "S", "CONFIDENTIAL": "C"},
    full_names={g: g for g in ["OPEN", "SENSITIVE", "CONFIDENTIAL"]},
    labels={"OPEN": "공개", "SENSITIVE": "민감", "CONFIDENTIAL": "기밀"},
    thresholds={"CONFIDENTIAL": 5.5, "SENSITIVE": 0.75, "OPEN": 0.0},
    entity_weights={}, keywords=[],
    zs_hypotheses=ZS_LABELS, exemplars=EXEMPLARS,
    bulk_pii_types=BULK_PII_TYPES, bulk_pii_threshold=10,
    floor_rules=[
        {"when": "bulk_pii", "min_grade": "CONFIDENTIAL", "code": "BULK-PII", "severity": "warn"},
        {"when": "entity", "entity": "JP_MY_NUMBER", "locale": "ja",
         "min_grade": "CONFIDENTIAL", "code": "JP-MyNumberAct", "severity": "error",
         "extra": "myNumberSuppressForced"},
    ],
    regulations={"ko": ["KR-PIPA"], "ja": ["JP-APPI"], "en": ["US-CCPA"], "zh-CN": ["CN-PIPL"]},
)
GRADE_PROFILES = {"n2sf": N2SF_PROFILE}
DEFAULT_PROFILE = "n2sf"
```

## 4. 결정론적 floor — 침범 불가 규칙

`floor_rules`는 **앙상블 직후** 적용, **등급을 올리기만** 한다.

```
파일 → extract_text → 텍스트
     → T1 규칙(정규식·deny-list) ─┐
     → T2 NER                     ├─ findings → _score → base_grade
     → 키워드 스캔                 ─┘
     → T3 신경망(opt, zeroshot)
     → _combine(앙상블)  ← 확률 tier는 escalate만
     → apply_floor(profile.floor_rules)  ← 결정론적 강제상향(하향 불가)
     → SHAP → JSON
```

```python
def apply_floor(grade, findings, locale, profile, ctx):
    rank = {g: i for i, g in enumerate(profile.grades)}
    forced = grade
    for rule in profile.floor_rules:
        hit = _floor_rule_hit(rule, findings, locale, ctx)
        if hit and rank[rule["min_grade"]] > rank[forced]:
            forced = rule["min_grade"]
            ctx.violations.append({"code": rule["code"], "severity": rule["severity"]})
    return forced
```

> **불변식:** `rank[apply_floor(...)] >= rank[base_grade_from_T1]`.

## 5. API 변경

`profile=` 파라미터 추가, 기본값 `"n2sf"`.

```python
def classify(file, ..., profile="n2sf") -> dict: ...
def classify_text(text, *, ..., profile="n2sf") -> dict: ...
```

- `_score`/`_combine`/`apply_floor`/`_shap_block`/`neural_infer`가 활성 프로필 인자 수령.
- 반환 JSON에 `"profile"` 추가. 우선순위: **base < 프로필 < weights override**.

## 7. 검증 — N²SF 1급, 나머지 경량

| 항목 | N²SF | 기타 업체 |
|---|---|---|
| 회귀 게이트 | 기존 출력과 동일(fixture 바이트 비교) | — |
| C-recall 하드 게이트 | 최상위 등급 recall ≥ 목표, 단일 오분류도 실패 | soft 경고 |
| floor 불변식 | 필수 | 필수 |

---

## 8. 예외 경로 — 전용 3-class 변형 (필요 시에만)

범용 zeroshot이 게이트 미달일 때만 업체 데이터로 N-class 헤드 파인튜닝 →
`kind:"finetuned"` 백엔드 추가 + 프로필 locked. **mdeberta-n2sf 가 이 경로의 N²SF판이다.**

---

## 9. 작업 순서 (DoD)

1. `GradeProfile` + `GRADE_PROFILES`/`DEFAULT_PROFILE`.
2. 모듈 상수(§1) → `N2SF_PROFILE` 이전, 전역 상수는 파생 별칭.
3. `_score`/`_combine`/`_shap_block` 순서기반 일반화.
4. `apply_floor` 신설(bulk·마이넘버 분기 대체).
5. `neural_infer` 프로필 가설/예시 수령(전역 직접참조 제거).
6. `--profile` 추가(기본 n2sf).
7. `load_profile()` + 예시 YAML.
8. 회귀 테스트(profile=n2sf 동일성).
9. floor 불변식 테스트.
10. JSON `profile` 필드 + 문서 갱신.

### 절대 금지
- 등급 개수·이름을 모델 가중치에 박지 말 것(예외 §8뿐).
- 확률 tier가 등급을 내리게 두지 말 것.
- 런타임 HF 허브 로드 금지(에어갭).
- N²SF 회귀 게이트를 깨는 변경 머지 금지.

# L5 다국어 적대공격 — 누락률 평가 (방법 & 예비결과)

> 일자: 2026-07-02 · 목적: 기밀·민감 내용을 **비한국어(영/중/일)** 로 표현했을 때 분류기가 **누락(유출)** 하는지 측정.
> 배경: `다국어_적대공격_해소방안.md`의 우려("한국어전용 모델은 외국어 기밀을 놓칠 수 있다")를 **수치로 검증**.

---

## 1. 평가셋 (heldout_l5.json)

- **72건** = 3언어(en/zh/ja) × 3등급(O/S/C) × 8건. 결정론적 생성(`harness/gen_l5.py`).
- **C(기밀)**: 언어무관 강식별자(유효 주민번호·카드번호·API/AWS키·여권) + 각 언어 기밀표지
  (예: `CONFIDENTIAL`, `机密/绝密`, `極秘/社外秘`) + 내부 프로젝트명.
- **S(민감)**: 단일 개인정보(이메일/전화) + 민감 맥락(인사·연락처).
- **O(공개)**: 개인정보·기밀표지 없는 순수 안내문.
- 라벨은 **정책 기반**(분류기 규칙 아님) → 정직 평가.

**예시(zh/C)**:
```
绝密
本备忘录包含与项目Omega相关的受限资料，未经授权不得披露。
所附标识：5212-3308-0516-9780        ← 유효 카드번호(언어무관)
所附标识：740227-26400971            ← 유효 주민번호(언어무관)
```

---

## 2. 지표

- **언어별 기밀 누락률(leak rate)** = C 문서를 C 미만(O/S)으로 오분류한 비율. **0이 목표**(누락=유출=최악).
- L5 macro-F1 = 외국어 O/S/C 전반 분별력(과대분류 포함 반영).
- **뉴럴-단독** 기준. 실서비스는 **T1 정규식 floor**가 강식별자를 **언어무관**으로 잡아 누락을 추가 차단.

---

## 3. 예비 결과 (n2sf-small, 뉴럴-단독)

| 모델 | L5 macroF1 | L5 C재현율 | 언어별 누락(en/zh/ja) | (참고) 한국어 macroF1 |
|---|---|---|---|---|
| n2sf-small (KoELECTRA-small) | 0.349 | **1.00** | **0 / 0 / 0** | 0.709 |

**해석**
- **누락 0%** — 한국어전용 계열(KoELECTRA)인데도 외국어 기밀을 **하나도 안 놓침**. 강식별자(주민번호·카드)가 언어무관이라 뉴럴도 C로 강하게 반응.
- **L5 macroF1 0.35로 낮음** = 외국어 **O/S를 과대분류**(불명 언어를 의심스럽게 높게 판정). → **안전하나 과하게 보수적**.
- 이는 우리 **안전 우선 기조와 일치**: 과소분류(유출)=0, 대신 과대분류 감수.

---

## 4. 전체 매트릭스 (실측 완료)

전 모델 일괄 평가(`harness/eval_l5.py`, CPU, 뉴럴-단독):

| 모델 | L5 macroF1 | L5 C재현율 | 누락 en/zh/ja | 한국어 macroF1 |
|---|---|---|---|---|
| n2sf-small | 0.349 | 1.00 | **0 / 0 / 0** | 0.709 |
| n2sf-base(기존) | 0.556 | 1.00 | **0 / 0 / 0** | 0.719 |
| **n2sf-base-kd(개선·승격)** | **0.730** | 1.00 | **0 / 0 / 0** | **0.796** |
| **n2sf-klue-large** | **0.805** | 1.00 | **0 / 0 / 0** | 0.872 |
| n2sf-xlmr-large | 0.556 | 1.00 | **0 / 0 / 0** | 0.917 |

**핵심 실측**
- **전 모델 비한국어 기밀 누락 0%(en/zh/ja)** — 강식별자(주민번호·카드)가 언어무관이라 뉴럴도 C로 강반응. **누락 방지는 크기·언어 무관 확보.**
- **가설 반증**: 다국어 XLM-R-large가 KLUE보다 L5에서 낮음(0.556 < 0.805). 예상과 반대.
  - 이유: **교사 라벨이 한국어 중심**이라 XLM-R의 다국어 용량이 증류에서 안 쓰임 → 외국어 O/S를 과대분류.
  - 즉 "백본이 다국어"라고 자동으로 다국어 분별이 좋아지지 않음 — **다국어 교사 라벨이 있어야** 발현.
- **균형샘플링 효과**: base 0.556 → base-kd 0.730. 한국어(0.719→0.796)뿐 아니라 **외국어 분별도 동반 개선**.
- L5 macroF1이 낮은 모델(small/xlmr)은 외국어 O/S **과대분류**(안전엔 무해, 정밀도만 손해).

---

## 5. 결론(실측) & 안전 설계

- 뉴럴-단독으로도 **비한국어 기밀 누락 0**(강식별자 효과). 외국어 O/S 과대분류는 있으나 안전엔 무해.
- **다층 방어**: ① T1 정규식(언어무관 강식별자·다국어 기밀표지 키워드) → ② 뉴럴 → 누락 이중 차단.
- **다국어 분별을 실제로 높이려면**: 백본만 다국어로 바꾸는 것으론 부족 → **다국어(영/중/일) 교사 라벨을 증류셋에 추가**해야 XLM-R 용량이 발현. (후속 과제)
- 값싼 즉효책: 다국어 기밀표지 키워드(机密/極秘/CONFIDENTIAL 등)를 `GRADE_KEYWORDS`에 추가하면 규칙 tier에서 언어무관 검출(누락 방지 추가 강화).

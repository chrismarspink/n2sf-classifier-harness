# 근본 성능 향상 방안 — 점수가 "오락가락"하는 이유와 진짜 해결책

> 일자: 2026-07-06 · 질문: 점수가 오락가락한다. 현 모델에서 근본적 성능 향상 방안이 있나?
> 핵심: 오락가락은 **모델보다 "측정과 데이터"가 불안정**하다는 신호. 근본책은 손잡이(앙상블·임계)가 아니라
> **① 신뢰할 평가 ② 실데이터 ③ 과제 정의**를 고치는 것.

---

## 1. 왜 오락가락하나 (진단)

| 원인 | 근거 | 효과 |
|---|---|---|
| **평가셋이 작음(72)** | 문서 1건 ≈ ±1.4%p | 에폭·시드마다 ±5%p 흔들림 |
| **분포에 따라 점수 급변** | 동일분포 0.97 vs OOD 0.65~0.74 | "점수"가 어느 셋이냐에 좌우 |
| **합성·동일 생성기** | LLM 문체 암기 → OOD 붕괴 | 진짜 실력 아님 |
| **라벨이 LLM/정책(순환)** | 사람 정답 없음 | 상한·정답 불명 |
| **best-save·시드·에폭 확률성** | C재현율 우선 저장 등 | 체크포인트마다 값 다름 |
| **손잡이 튜닝(앙상블·규칙 floor)** | floor 추가 시 0.958→0.838 | 안전↔정밀 트레이드오프로 진동 |

→ **결론: 지금 "점수"는 노이즈가 큰 자(ruler)로 잰 값.** 자를 고치기 전엔 개선을 개선으로 확인 못 함.

---

## 2. 근본 향상 방안 (레버리지 순)

### ★ 1. 측정을 신뢰 가능하게 (가장 근본 — "못 재면 못 고친다")
- **사람 라벨 골드셋 500+**(등급×언어×포맷 층화) + **평가자 간 일치도(κ)** 측정.
- **평가셋 확대(72→300~500)** + **부트스트랩 신뢰구간** 보고(점추정 금지).
- **다중 분포**(동일·OOD·실문서)로 **구간** 보고.
- 효과: 오락가락의 상당 부분이 **측정 노이즈**였음을 제거 → 진짜 개선만 남음.

### ★ 2. 실데이터로 (합성→실 갭이 OOD 붕괴의 주범)
- **도입처 실문서(비식별)** 를 학습·평가에 편입 — 배포 분포에 직접 정렬.
- **준지도**: 실 미라벨 문서를 교사 앙상블로 라벨 → 불일치만 사람 확정(능동학습).
- 효과: OOD/실서비스 성능의 **근본 상승**(현 0.7대 → 실데이터 정합 시 상향).

### ★ 3. 과제 정의·라벨 품질 (상한 자체를 올림)
- **등급 기준서(rubric) 정교화**: S↔O, S↔C 경계 규칙 명문화. 교사끼리 40%+ 갈리면 **모델이 아니라 기준이 모호**.
- **다교사 합의 + 사람 판정**(경계 문서): 정답지 일관·정확 → 학생 상한↑.
- 효과: 모호성으로 인한 천장(0.9x) 자체를 상향.

### 4. 학습 방법 (과적합·분산 축소 — 보조적 근본)
- **도메인 적응 사전학습(DAPT)**: 고객 도메인 미라벨 코퍼스로 인코더 **추가 사전학습** 후 파인튜닝 → 도메인 정합(강력).
- **다중 생성기·강증강**(진행 중 #1) + **일관성 정규화/적대학습**.
- **체크포인트 평균(SWA)·다시드 앙상블** → **분산(오락가락) 직접 감소**.
- **캘리브레이션 + 저신뢰 abstain/사람검토** → 원점수 흔들려도 운영 품질 안정.

### 5. 규모·구조 (수확 체감이나 실효)
- 라지 백본(이미 강건성 우위 확인) 유지·확대. 장문은 maxlen↑/RAG.

---

## 3. 손잡이 vs 근본 (구분)

| 손잡이(지금까지) | 근본(다음) |
|---|---|
| 앙상블 가중·임계·규칙 floor | 신뢰 평가(사람 골드셋·CI) |
| best-save 기준·에폭 | 실데이터 편입 |
| 백본 교체 | 과제 rubric·합의 라벨 |
| — | 도메인 적응 사전학습·다시드 앙상블 |

- 손잡이는 **트레이드오프 이동**(안전↔정밀)일 뿐 상한을 못 올림 → 오락가락의 원인이기도.
- 근본은 **입력(데이터·라벨·과제)과 자(평가)** 를 바꿔 상한과 안정성을 동시에 올림.

---

## 4. 권고 (우선순위·자원)

1. **지금 당장(자동 가능)**: 평가셋 확대(300~500 합성) + **부트스트랩 CI** + **다시드 학습·SWA**로 분산 축소.
   → "오락가락"을 수치로 잡고, 개선을 신뢰성 있게 측정. (내가 바로 실행 가능)
2. **사람 필요(핵심)**: **골드셋 500 라벨링**(지침·후보추출은 내가 준비) → 절대 기준·SLA.
3. **데이터 확보 시(최대 효과)**: 도입처 **실문서** 편입 + DAPT.
4. 병행: 다중 생성기(#1 진행 중)·규칙 floor(#2 완료)로 강건·안전 보강.

> 요약: **현 합성·72문서 체제에선 추가 튜닝은 대부분 노이즈**입니다. 근본 향상은
> **평가를 신뢰 가능하게(사람 골드셋·CI·다시드) → 실데이터·rubric으로 상한을 올리는** 순서입니다.

---

## 5. 즉시 실행 제안
- **A. 평가 안정화 패키지**(자동): held-out 300~500 생성 + 부트스트랩 CI + 다시드(3seed) 재학습·SWA → 분산·상한 재측정.
- **B. 골드셋 준비**: 라벨링 지침 + 층화 후보 500건 추출(사람 라벨 대기).
- **C. (#1 완료 후)** 실데이터 편입 파일럿 설계.

> 어느 것부터 진행할지 지시 주시면 A는 즉시 착수합니다(오락가락을 CI로 먼저 잡는 것을 추천).

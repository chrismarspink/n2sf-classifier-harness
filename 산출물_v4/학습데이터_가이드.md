# 온사이트 학습 데이터 가이드 (v4) — 등급별 몇 개가 필요한가?

> 질문: 등급별 **한 파일만** 학습해도 되나? **특정 개수 이상** 필요? **모든 등급 동등 수**여야 하나?
> 전제: 우리는 ① 기존 배포 모델에서 **이어 학습(전이)** ② 기존 증류셋과 **혼합(망각 방지)** ③ **균형 샘플링**.
> 이 3장치 때문에 소량·불균형에도 **모델이 망가지지 않지만**, 개수·균형에 따라 **효과**가 달라진다.
> (관리 콘솔 「데이터 관리」 업로드 시 아래 판정·권고가 자동 표시된다 — admin_site.py `check_dataset`)

---

## 1. 한 줄 답

- **한 등급만**(예: C만) → ❌ **금지**(모델이 그 등급으로 붕괴). **최소 2개, 권장 3개 등급 모두** 포함.
- **등급당 1개 파일** → ⚠️ **손상은 없으나(혼합 덕분) 학습효과 미미**. "적용해도 안전하지만 거의 안 변함."
- **의미 있는 향상** → 등급당 **≥50건**(권장 **100~300건**).
- **균형** → **완전 동수일 필요 없음**(균형샘플러가 보완). 단 **최소 등급 ≥ 최다 등급의 20%** 유지 권장.

---

## 2. 개수별 효과 (판정표 — 관리콘솔 검증과 동일)

| 상황 | 판정 | 효과 |
|---|---|---|
| 1개 등급만 | **불가(error)** | 붕괴 위험 → 학습 차단 |
| 등급당 1~9건 | 부족(warn) | 전이+혼합으로 **소폭 보정**만. 사실상 기존모델 유지 |
| 등급당 10~49건 | 가능 | 경계(S↔O 등) **부분 개선** |
| **등급당 50~300건** | **권장(ok)** | **의미 있는 향상**(고객 도메인 정합) |
| 최다:최소 > 5배 | 불균형(warn) | 균형샘플러 보완하나 **소수 등급 보강 권장** |

---

## 3. 왜 그런가 (근거)

- **전이학습**: 백지가 아니라 **이미 잘하는 모델에서 미세조정** → 소량으로도 효과. 그래서 "수만 건" 불필요.
- **혼합(mix)**: 고객 데이터만 쓰면 **파국적 망각**(기존 능력 상실) → 기존 증류셋을 섞어 방지. 그래서 **1건/등급도 안전**(대부분 기존셋으로 학습).
- **균형 샘플링**: 학습 시 등급을 **등확률로 추출** → 개수 불균형을 상당 부분 상쇄. 그래서 **완전 동수 불필요**. 단 소수 등급이 **다양성**을 못 채우면 그 등급 일반화는 약함.

---

## 4. 등급별 실무 권고

- **C(기밀)**: 실무상 **가장 희소**. 억지로 많이 만들 필요 없음 —
  **기밀표지(기밀/대외비/극비) 규칙 floor + 혼합셋**이 C 안전(누락0)을 보장. 있으면 **10~50건**으로 충분.
- **S(민감)·O(공개)**: **고객 문서가 기존과 가장 다른 구간**(문체·양식·경계). 여기에 **라벨링을 집중**(각 100~300건)하면 향상 큼.
  특히 v4 기본 모델(n2sf-official)은 합성 §9 데이터라 **O를 C로 과대분류하는 경향**이 있어 → **실제 O(공개)문서 라벨을 넉넉히** 넣으면 과대분류가 크게 줄어든다.
- 즉 **"C 조금 + S/O 넉넉히(특히 실제 O), 대략 균형"** 이 가성비 최적.

---

## 5. 권장 레시피 (기본값)

```
등급당 목표: C ≥ 30, S ≥ 100, O ≥ 100  (합 ≥ 250)
불균형이면: mix-ratio(기존셋 혼합) 0.5~0.7 로 올려 안정화
1차 소량(등급당 20~30)으로 파일럿 → 관리콘솔에서 held-out 확인 → 부족하면 라벨 추가
```

- 관리콘솔: 데이터 관리(업로드·검증) → 학습(데이터셋 선택) → 자동 무중단 적용 → 모델 버전(롤백 가능).
- **안전 불변**: 재학습과 무관하게 §9 규칙 floor(기밀표지→C, 개인정보→S)는 항상 적용 → 재학습이 안전을 깨지 않음.

---

## 6. 결론
- **한 등급만 = 불가. 등급당 1개 = 안전하나 무의미. 권장 등급당 50~300, 대략 균형.**
- 완전 동수는 불필요(균형샘플러). C는 희소해도 규칙·혼합이 커버 → **S/O(특히 실제 O 공개문서) 라벨링에 집중**이 효율적.

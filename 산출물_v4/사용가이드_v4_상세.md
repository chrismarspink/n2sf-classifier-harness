# N²SF classifier_v4 상세 가이드 — 반환 JSON · 확장 변수 · GUI 렌더링

> 목적: **판단 근거 전체를 노출**하는 v4 함수의 (1) 반환 JSON 스키마 전 필드, (2) 정규식·키워드·§9 확장 변수,
> (3) 화면(GUI)에서 "왜 이 등급인지" 그리는 법. 사람·AI·프론트엔드 개발자 공용.
> 참고: 엔진 상세는 `data_classifier.py` / `규칙_정책_확장_방안.md`.

## 1. 호출

```python
from classifier_v4 import N2SFExplainClassifier
clf = N2SFExplainClassifier(model="accurate")     # 티어 또는 백엔드/경로
r = clf.classify("문서.pdf")                       # 파일 경로 또는 원문 텍스트(is_file=False)
# r 은 아래 §2 상세 JSON. GUI가 그대로 렌더 가능.
clf.reload_model("cust-v2", path="models/n2sf-custom-v2")   # 무중단 교체(v3와 동일)
```
CLI: `python classifier_v4.py 문서.pdf --model accurate` · `--demo`

## 2. 반환 JSON 스키마 (전 필드)

```jsonc
{
  "schema": "n2sf-explain-v4",
  "grade": "C|S|O",                 // 최종 등급 코드
  "gradeLabel": "기밀|민감|공개",
  "confidence": 0.0-1.0,            // 신뢰도
  "score": 7.0,                     // 규칙 점수(엔티티 가중합)

  "file": {                         // 파일 메타(GUI 헤더)
    "source": "/path 또는 text",
    "format": "pdf|docx|xlsx|hwpx|pptx|csv|md|json|text",
    "sizeBytes": 12345, "sha1_12": "ab12…",   // 파일일 때
    "extractedChars": 1024          // 추출 텍스트 길이
  },

  "decision": {                     // 판정 요약(GUI 상단 배지)
    "finalGrade": "C",
    "decidedByTier": "T1(규칙) | T3(뉴럴)+앙상블",
    "earlyExit": true,              // 규칙에서 기밀 확정 → 뉴럴 생략
    "n2sfPrinciple": "명시적 기밀표지 → C(§9 비밀/보안업무규정)",  // 적용된 §9 원칙
    "reason": "기밀/대외비/극비 … 검출 → 기밀 하드 floor."
  },

  "tiers": {                        // 티어별 판정(파이프라인 시각화)
    "rules":   {"grade","confidence","version"},
    "ner":     {"grade","confidence","version"},
    "neural":  {"grade","confidence","scores":{O,S,C},"backend","version"},
    "ensemble":{"grade","method","weights","probs":{O,S,C}}
  },

  "findings": {                     // 근거 상세(GUI 근거 테이블)
    "regexEntities": [ {"entity":"KR_RRN","text":"900101-…","start","end","score","recognizer"} ],  // 강식별자(정규식/deny-list)
    "nerEntities":   [ {"entity":"PERSON","text":"홍길동","start","end","score","recognizer"} ],      // 문맥 개체(spaCy NER)
    "keywords":      [ {"type":"KEYWORD_SECRET","label":"대외비","count","weight","spans","samples"} ],// 등급 키워드
    "piiByType":     [ {"type","label","count","source","weight","spans","samples"} ],                // 집계(점수 기여)
    "piiTotal": 3
  },

  "shap": {                         // 왜 이 등급(가법적 기여도) — GUI 막대차트
    "baseline": 0.0, "total": 7.0,
    "contributions": [ {"feature":"KR_RRN","label":"주민등록번호","count":1,
                        "contribution":2.0,"percent":0.29,"direction":"up"} ]
  },

  "compliance": {                   // 규제·위반(감사)
    "regulations": ["KR-PIPA"],
    "violations": [ {"code":"KR-PIPA-Art23","msg":"…","severity":"warn"} ]
  },

  "explanation": "이 문서는 기밀(C)…근거…원칙…",   // 사람이 읽는 한 문장(GUI 상단)
  "modelVersion": "accurate#v3",
  "elapsedMs": 42
}
```

**필드 요약**: `grade/gradeLabel/confidence`(결과) · `decision.n2sfPrinciple`(적용 원칙) ·
`findings.regexEntities/nerEntities/keywords`(무엇이 잡혔나) · `shap.contributions`(각 근거 %) ·
`explanation`(요약문) · `tiers`(어느 티어가 결정) · `file`(파일 정보) · `compliance`(규제).

## 3. 확장 변수 — 정규식/키워드/§9 조정 (엔진 `data_classifier.py`)

### 3.1 새 정규식(강식별자) 추가 — `PATTERN_RECOGNIZERS`
```python
PATTERN_RECOGNIZERS.append({
  "name": "KR_HEALTH_INS", "supported_entity": "KR_HEALTH_INS",
  "patterns": [{"name":"hi","regex": r"\b\d-\d{10}\b","score":0.6}],
  "context": ["건강보험","요양기관"]})        # 문맥어 있으면 점수↑
ENTITY_WEIGHTS["KR_HEALTH_INS"] = 2.0        # S 대역(§9: 개인정보=S)
data_classifier._LOCALE_ENTS["ko"].add("KR_HEALTH_INS")   # NER slim 통과(최적화)
```

### 3.2 등급 키워드 추가 — `GRADE_KEYWORDS` / 런타임 `weights`
```python
GRADE_KEYWORDS.append(("영업비밀", 2.0, "영업비밀"))       # 코드 추가(S 기여)
# 또는 무재배포(런타임): clf.classify(text, ...) 대신 dc.classify_text(text, weights={"keyword":[
#   {"keyword":"영업비밀","weight":2.0,"label":"영업비밀"}]})
```

### 3.3 명시적 기밀표지(하드 floor=C) — `SECRET_FLOOR_KW`
```python
SECRET_FLOOR_KW.add("한limited")            # 이 표지 검출 시 무조건 C(§9 비밀). 언어무관.
GRADE_KEYWORDS.append(("한limited", 3.0, "한limited"))
```

### 3.4 §9 등급 매핑 조정 — 가중치/임계
| 변수 | 의미 | §9 기본 |
|---|---|---|
| `ENTITY_WEIGHTS[type]` | 엔티티 점수 기여 | 개인식별자=2.0(S 대역), C 아님 |
| `C_THRESHOLD` / `S_THRESHOLD` | 등급 임계 | 5.5 / 0.75 (규칙 등급 상한=S) |
| `SECRET_FLOOR_KW` | C 하드 floor 표지 | 기밀/대외비/극비/機密/… |
| `BULK_PII_THRESHOLD` | 대량 개인정보 → S floor | 10건 |
| `_LOCALE_ENTS[locale]` | NER 최적화 대상 엔티티 | 로케일별 |

### 3.5 런타임 오버라이드(무재배포) — `weights` 인자
```python
dc.classify_text(text, weights={
  "entity": {"KR_RRN": 3.0},                  # 가중치 즉시 조정
  "keyword": [{"keyword":"이사회","weight":2.0,"label":"이사회"}],
  "thresholds": {"confidential":5.0,"sensitive":0.7},
  "tier": {"rules":0.3,"ner":0.3,"neural":4.0}})
```
> 고객사별 정책은 `GradeProfile`(YAML) 설계로 외부화 가능 — `등급프로필_외부화_명세.md`.

## 4. GUI 렌더링 가이드 ("왜 이 등급" 화면)

| 화면 영역 | 사용 필드 |
|---|---|
| **상단 배지** | `gradeLabel`+`grade` 색상(C=적/S=황/O=녹), `confidence` |
| **한줄 설명** | `explanation` |
| **적용 원칙** | `decision.n2sfPrinciple` + `reason` (§9 근거) |
| **근거 막대차트** | `shap.contributions`(label×percent) — 각 근거가 등급에 몇 % 기여 |
| **검출 테이블** | `findings.regexEntities`(강식별자) / `nerEntities`(인물·기관) / `keywords`(표지) — text·위치(span)로 원문 하이라이트 |
| **파이프라인** | `tiers`(rules→ner→neural 각 grade) + `decision.earlyExit`(뉴럴 생략 여부) |
| **파일 정보** | `file`(포맷·크기·해시) |
| **규제/감사** | `compliance.violations` |

- **원문 하이라이트**: `spans`(start,end)로 검출 위치를 원문에 색칠 → 사용자가 "여기 주민번호 때문에 S" 직관 이해.
- **설명 가능성**: 규칙·키워드·SHAP 기여도가 그대로 근거 → LLM 블랙박스 대비 강점.

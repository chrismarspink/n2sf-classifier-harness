# N²SF classifier_v4 상세 가이드 — 단일 파일 3-tier(연구소 표준) · GUI/XAI

> `classifier_v4.py` **한 파일**에 3-tier 전체(규칙 정의 포함)가 들어있다. 연구소가 이 파일을 표준으로 편집·배포.
> 추론 100% 로컬(외부 LLM/GPU 0). **기본 모델=§9 학습판(n2sf-official)**. 반환은 GUI/XAI용 상세 JSON.
> 학습/핫스왑은 v3 준용(`train_onsite.py` + `reload_model`).

---

## 1. 단일 파일 구조 (편집 지점이 파일 안에 있음)

`classifier_v4.py` 상단 = **규칙 정의(연구소가 직접 편집)**:
| 변수 | 역할 |
|---|---|
| `PATTERN_RECOGNIZERS` | 정규식 인식기(주민번호·카드·키 등) — dict 추가로 즉시 등록 |
| `DENY_LIST_RECOGNIZERS` | 고정 어휘(사내 프로젝트·부서·VIP) |
| `GRADE_KEYWORDS` | 등급 키워드(기밀표지 등) `(키워드, 가중치, 라벨)` |
| `SECRET_FLOOR_KW` | 명시적 기밀표지 → **C 하드 floor** |
| `ENTITY_WEIGHTS` | 엔티티 점수 기여(개인정보=2.0=S 대역) |
| `C_THRESHOLD`/`S_THRESHOLD` | 등급 임계 |
| `_LOCALE_ENTS` | NER 최적화 대상(로케일별) |

중단부 = 엔진(추출·Presidio/spaCy NER·점수·§9 floor·앙상블·SHAP·뉴럴). 하단 = v4 설명층·(옵션)Fable·핫리로드.
→ **규칙 추가/수정은 이 파일 상단만 고치면 전 파이프라인 반영.**

## 2. 3(+1) 티어
- **T1 규칙**: 정규식·deny-list·키워드. 기밀표지 검출 시 **early-exit(즉시 C)**.
- **T2 NER**: Presidio+spaCy(오프라인). 인물·기관·지명.
- **T3 뉴럴**: 증류 모델(오프라인, 배치). O/S/C 확률.
- **T4 Fable5(옵션)**: `use_fable=True` + `N2SF_FABLE=1` + `ANTHROPIC_API_KEY` 일 때만. 기본 off(온프레미스 유지).
  경계 문서 보조 판정에 클라우드 LLM 참고(데이터 반출 발생 — opt-in).

## 3. 호출

```python
from classifier_v4 import N2SFExplainClassifier
clf = N2SFExplainClassifier()                 # 기본 model="n2sf-official"(§9)
r = clf.classify("문서.pdf")                   # 파일 경로 또는 원문(is_file=False)
clf.reload_model("cust-v2", path="models/n2sf-custom-v2")   # 무중단 교체(v3 준용)
clf2 = N2SFExplainClassifier(use_fable=True)   # (옵션) Fable 티어 사용
```
CLI: `python classifier_v4.py --demo` · `python classifier_v4.py 문서.pdf --model n2sf-official [--fable]`
모델 티어: `fast-ko / compact-multi / balanced / accurate / n2sf-official(기본·§9)` 또는 경로.

## 4. 반환 JSON (GUI/XAI) — 주요 필드

```jsonc
{ "grade":"C|S|O","gradeLabel","confidence","score",
  "file":{source,format,sizeBytes,sha1_12,extractedChars},
  "decision":{finalGrade,decidedByTier,earlyExit,n2sfPrinciple,reason},   // §9 원칙
  "tiers":{rules,ner,neural,ensemble},
  "findings":{regexEntities[],nerEntities[],keywords[],piiByType[],piiTotal},
  "shap":{baseline,total,contributions:[{label,contribution,percent}]},    // 왜 이 등급
  "compliance":{regulations,violations},
  "explanation":"사람이 읽는 요약",
  "fableTier":{grade,raw}?,   // use_fable 시
  "modelVersion","elapsedMs" }
```
전 필드·GUI 렌더(배지·SHAP 막대·검출 하이라이트·파이프라인)는 §6.

## 5. 정규식/키워드 추가 예시 (이 파일 상단 편집)

```python
# 새 정규식(건강보험번호) — §9: 개인정보이므로 S 대역
PATTERN_RECOGNIZERS.append({"name":"KR_HEALTH_INS","supported_entity":"KR_HEALTH_INS",
  "patterns":[{"name":"hi","regex":r"\b\d-\d{10}\b","score":0.6}],"context":["건강보험"]})
ENTITY_WEIGHTS["KR_HEALTH_INS"] = 2.0
_LOCALE_ENTS["ko"].add("KR_HEALTH_INS")            # NER 최적화 통과

# 사내 프로젝트/부서(deny-list)
DENY_LIST_RECOGNIZERS.append({"name":"INTERNAL_PROJECT","supported_entity":"INTERNAL_PROJECT",
  "deny_list":["프로젝트오로라","전략기획실"]})

# 등급 키워드 / 기밀표지(C 하드 floor)
GRADE_KEYWORDS.append(("영업비밀",2.0,"영업비밀"))
SECRET_FLOOR_KW.add("한limited")                   # 이 표지 → 무조건 C
```
> 무재배포 런타임 조정은 `classify_text(..., weights={"entity":{...},"keyword":[...],"thresholds":{...}})`.

## 6. GUI 렌더링

| 영역 | 필드 |
|---|---|
| 등급 배지 | `gradeLabel`+`grade`(C적/S황/O녹), `confidence` |
| 한줄 설명 | `explanation` |
| 적용 원칙 | `decision.n2sfPrinciple`+`reason` |

| 파이프라인 | `tiers`(rules→ner→neural) + `decision.earlyExit` |
| 파일·규제 | `file`, `compliance.violations` |

## 7. 학습 & 핫스왑 (v3 준용)
- **재학습**: `train_onsite.py`(고객 라벨→전이+혼합→새 버전). 상세 `../산출물_v3/온사이트학습_핫리로드_가이드.md`.
- **무중단 교체**: `clf.reload_model(name, path=...)` — 새 모델 사전 로드 후 원자적 스왑, 다운타임 0, 실패 시 기존 유지.

## 8. 정확도·한계 (정직)
- **기본 n2sf-official = §9 학습판**(개인정보=S). 단 합성 §9 데이터(400건) PoC라 **O→C 과대분류 경향**(§9 held-out: macroF1 0.788, C재현율 1.0, 과대 0.20). 안전측이나 **실문서 재학습 필요**.
- 다른 티어(accurate/fast-ko 등)는 **구 rubric(개인정보=C)** 학습 → §9엔 official 사용 권장.
- SHAP 없는 뉴럴-C의 `n2sfPrinciple`은 "국가안보 추정"으로 표기 → **검토 큐 권장**(과대분류 흡수).

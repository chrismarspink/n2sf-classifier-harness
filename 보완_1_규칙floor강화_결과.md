# 보완 #2 — 규칙 floor 강화 결과 (OOD 누출 봉쇄)

> 일자: 2026-07-06 · 목적: OOD에서 발생한 기밀 누출(base-ml 0.948, small-v2 0.780)을 규칙 floor로 차단.
> 결과: **추천 라지 4종 OOD 기밀재현율 1.00 달성**, small-v2 0.780→0.994. 비용=과대분류 ↑(안전-우선상 허용).

---

## 1. 진단으로 찾은 누출 원인(OOD)

1. **전각 숫자 주민번호**: `９７１２１１-１６２２４０５１` → 정규식 `\d{6}-\d{7}` 불일치 → 규칙 검출 0.
2. **띄어쓴 기밀표지**: `대 외 비`, `기 밀` → 키워드 "대외비/기밀" 불일치 → 검출 0.
3. **명시적 기밀표지를 앙상블이 하향**: `[극비]` 검출(규칙 8.0)해도 뉴럴이 S로 강하면 소프트 앙상블이 S 채택.

---

## 2. 적용한 수정 (`data_classifier.py`)

- **전각→반각 정규화**(`_normalize_obfuscation`): 전각 숫자·하이픈을 반각으로 → 전각 주민번호/카드 검출.
- **공백제거 키워드 매칭**: 고위험 표지는 공백 제거본에서도 탐지 → `대 외 비` 포착.
- **명시적 기밀표지 하드 floor**: 극비/대외비/기밀/사외비/機密/極秘/绝密/机密/社外秘/confidential/top secret 검출 시
  **CONFIDENTIAL 강제**(앙상블 하향 불가).
- 다국어 표지 보강(極秘/绝密/사외비 등). 과탐 유발 흔한어(비밀/보안/秘密)는 제외.

## 4. 남은 잔여 누출 (base-ml·small-v2)

- 잔여 1~2건은 **식별자·표지 없이 문맥상으로만 기밀**인 문서 → 규칙도 못 잡음.
- 대응: **보완 #3(저신뢰 시 상향/사람검토)** 또는 **#1(다중생성기 재증류)로 뉴럴 강건화**.
- 실무: **추천 모델을 라지(xlmr)로** 하면 이미 누출 0.

## 5. 결론
- **안전(누출0) 목표를 규칙 floor로 크게 달성**(라지 4종 1.00). 정규화·하드floor는 값싸고 언어무관.
- 과대분류 상승은 안전 비용 — 다음 **#1(근본: 학습 분포 다양화)** 으로 뉴럴을 강건화해 과대분류를 되돌리는 게 목표.

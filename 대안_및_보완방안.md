# OOD 과적합·안전 붕괴 — 대안 및 보완 방안

> 일자: 2026-07-06 · 배경: OOD 재평가에서 실성능 0.65~0.74, base-ml·small은 OOD에서 기밀 누출.
> 목표: (1) OOD/실문서 성능↑ (2) **기밀 누출0을 전 모델·OOD에서 보증** (3) 정직한 SLA.

---

## 1. 원인 재정의

- **근본 원인 = 학습 분포가 편협**: 학습 문서가 주로 한 생성기(LLM) 문체 → 그 분포엔 강하나 다른 분포(템플릿/실문서)에 약함.
- **안전(누출)의 원인**: 소형/base 뉴럴이 낯선 문서의 **의미상 기밀**(강식별자 없는)을 놓치고, 규칙 floor도 못 잡음 → 누출.
- **관찰**: 라지(용량↑)만 OOD에서 강건·누출0 → 용량과 데이터 다양성이 열쇠.

---

## 2. 대안 (선택 관점)

| 대안 | 내용 | 트레이드오프 |
|---|---|---|
| **A. 라지 기본 채택** | 안전·강건성 확보 위해 xlmr-large를 기본 티어로 | 용량 2.3GB·지연↑ (허용 가능) |
| **B. 소형은 "1차 필터"로만** | 소형은 명백 O 걸러내기용, 최종 등급은 라지/규칙 | 2단 파이프라인 복잡도 |
| **C. 라지+규칙 이중게이트** | 라지 뉴럴과 규칙 중 **높은 등급 채택**(보수) | 과대분류↑(검토 큐로 흡수) |
| **D. 앙상블(다양 백본)** | 서로 다른 백본·생성기로 학습한 모델 투표 | 자원↑, 이득 불확실 |

> 권고: **A+C 기본**(라지 + 보수적 이중게이트), 경량이 꼭 필요한 곳만 B.

---

## 3. 보완 방안 — 4개 레버

### 레버 1 · 데이터 다양화 (근본 해결, 최우선)
- **다중 생성기 학습**: gpt-4o + o4-mini + gpt-5.4 + **Gemma(로컬)** + **템플릿(corpus.py)** 을 섞어 학습셋 생성
  → 한 문체 과적합 방지. (OOD 격차의 직접 원인 제거) ★ **가장 값싼 근본책**
- **실문서 계열 편입**: 실제 공문서/HWPX 서식·뉴스·공개보고서(비식별) + synth PII 주입.
- **증강**: 패러프레이즈·역번역·포맷 변형(표/목록/머리말)·난독(띄어쓰기/완곡)으로 스타일 암기 차단.
- **경계·적대 강화**: S↔O, S↔C 경계 및 L4/L5 난이도 대량 추가.

### 레버 2 · 강건 학습 기법
- **라지 백본 우선**(이미 강건성 입증). 정규화(weight decay·dropout·R-drop), **OOD성 검증셋 기준 early-stop**.
- **혼합 증류**: 여러 교사(o4-mini 주 + gpt-4o 검산) 합의 라벨 + 불일치=경계 표시(능동학습).
- **캘리브레이션**: 온도 보정으로 신뢰도 신뢰화 → 임계 기반 판단.

### 레버 3 · 안전망 (누출0 보증, 즉효)
- **규칙 floor 강화(T1)** ★ **즉효·값쌈**: 정규식·키워드·deny-list 확장(다국어 기밀표지 机密/極秘/CONFIDENTIAL,
  더 많은 식별자·사내어휘) → 뉴럴이 놓쳐도 규칙이 기밀 포착. OOD 누출 직접 감소.
- **저신뢰 시 보수 처리(abstain/escalate)**: 뉴럴 신뢰도 낮음 또는 티어 불일치 → **상향(보수) + 사람 검토 큐**.
  "누출"을 "과대분류+검토"로 전환.
- **OOD 감지**: 입력이 학습분포에서 멀면(최대확률↓/엔트로피↑/임베딩 거리↑) **엄격 정책** 적용(등급 상향·검토).
- **강식별자·대량PII floor**(기존) 커버리지 확대.

### 레버 4 · 검증·운영 (정직한 SLA)
- **사람 라벨 골드셋 200~500** → 절대 기준·SLA 근거(누락률 보증).
- **다중 OOD**(템플릿·타 LLM·실문서)로 **구간 보고**(점추정 금지) + 부트스트랩 CI.
- **슬라이스 지표**(등급×언어×포맷×난이도)로 약점 국소화.
- **운영 모니터링**: 분포드리프트 감지 + 표본 감사 + 재보정 루프.

## 4. 즉시 실행 제안 (효과 검증형)

| 순위 | 실험 | 기대 | 소요 |
|---|---|---|---|
| **1** | **다중 생성기 학습셋**(LLM+Gemma+템플릿)으로 xlmr 재증류 → **새 OOD로 재평가** | OOD 격차 축소 입증 | 반나절(생성+증류) |
| **2** | **규칙 floor 강화** 후 전 모델 OOD **C재현율** 재측정 | OOD 누출↓(특히 소형/base) | 1~2시간 |
| **3** | **저신뢰 abstain→상향** 로직 추가 후 OOD 누출/과대 재측정 | 누출0 전환 | 1~2시간 |

> 1은 근본책(데이터), 2·3은 안전망(즉효). 셋 다 **재평가로 효과를 수치 검증**하는 게 핵심 —
> 이번처럼 "동일분포 착시"를 피하려면 **반드시 별도 OOD로 확인**.

---

## 5. 배포 자세 (지금 당장 적용)

- **안전-critical 등급판정 = 라지**(xlmr). 소형은 보조/사전필터.
- **C재현율(누락률)을 SLA 지표로**(F1 아님). 과대분류는 **사람 검토 큐**로 흡수.
- **고객 실문서 재보정 전 = 파일럿**으로만. 대외 수치는 **구간(0.7~0.9) + OOD 검증 예정** 표기.
- 고신뢰 O만 자동승인, C/S 경계는 human-in-the-loop.

---

## 6. 결론
- **대안**: 라지 기본 + 보수적 이중게이트(A+C).
- **보완**: ①다중생성기 데이터(근본) ②규칙 floor 강화·저신뢰 상향(안전 즉효) ③사람 골드셋·다중 OOD(검증).
- 모든 개선은 **별도 OOD 재평가로 효과 확인** — 동일분포 수치는 믿지 않는다.

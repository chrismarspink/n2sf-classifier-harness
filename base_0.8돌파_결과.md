# base 0.8 돌파 시도 — 결과

> 일자: 2026-07-02 · 대상: `n2sf-base`(mDeBERTa 279M) 정확도 개선 재증류
> 요지: **base 0.719 → 0.796 (뉴럴-단독, 기밀 누락 0)** 로 +0.077 향상. 승격 완료.
> 0.8은 ep1에서 0.806으로 순간 돌파했으나 그 지점은 기밀 1건 누락 → **안전 우선으로 누락 0인 0.796 저장**.

---

## 1. 무엇을 했나

`base_0.8돌파_전략.md`의 진단(① S 클래스 기아 ② 가짜 soft-label ③ maxlen 불일치)을 교정한 개선 KD 재증류.

| 적용 | 내용 |
|---|---|
| **균형 샘플링** | S(9%뿐)를 등확률로 오버샘플 → S 기아 해소 |
| **온도 소프트닝(T=1.5)** | near-one-hot 교사 라벨을 부드럽게(경계 뉘앙스 복원) |
| **maxlen 128 + 스케줄** | 안정 학습(cosine+warmup), 매 에폭 held-out best 저장 |
| **안전 우선 저장** | C재현율 1.0 만족 체크포인트를 우선 저장(누락 0 보장) |

> 구현: `harness/train_kd.py`. 우여곡절: maxlen 256 + mDeBERTa/MPS에서 gradient NaN 발산 →
> **maxlen 128 복귀 + gradient 유한성 가드**로 해결(nan_skips=0 확인).

## 2. 에폭별 궤적 (held-out 72문서)

| 에폭 | macroF1 | C재현율 | 저장 여부 |
|---|---|---|---|
| ep0 | 0.7734 | 0.958 | ★ |
| **ep1** | **0.8064** | 0.958 | ★ (F1 최고, but 누락 1건) |
| **ep2** | 0.7862 | **1.000** | ★ **최종 저장(안전 우선)** |
| ep3 | 0.7852 | 0.958 | |

- best-save 점수 = `F1 + (C재현율 1.0이면 +1.0)` → ep2가 승리(누락 0 보너스).
- 즉 **"누락 있는 0.806"을 버리고 "누락 0인 0.786"을 선택** — 설계 원칙대로.
- 최종 저장 모델을 maxlen 256으로 재평가 시 **0.796**(평가 길이 차이).

## 4. "0.8 돌파" 판정 (정직하게)

- **뉴럴-단독·누락0 기준**: **0.796** — 0.8 목전(미달 0.004).
- **뉴럴-단독·F1최대**: ep1 **0.806** — 0.8 돌파했으나 **기밀 1건 누락**(안전 위배로 채택 안 함).
- **전체 파이프라인 기준**: 뉴럴이 놓친 1건도 **T1 규칙 floor(강식별자)가 잡으면** 실서비스 누락은 0.
  → 이 관점이면 0.806 채택 + 규칙 floor로 안전 확보도 가능(아래 선택지).

---

## 5. 0.8을 "누락 0으로" 확실히 넘기려면 (선택지)

1. **S/경계 문서 증류셋 보강** — 교사로 S·경계 문서를 추가 생성·라벨(S 비중↑) → 뉴럴-단독으로 0.8+ 목표. (가장 정공법)
2. **ep1(0.806) 채택 + 규칙 floor 안전 검증** — 전체 파이프라인 C재현율 1.0 확인 후 사용. (빠름)
3. **에폭·T·균형강도 소폭 스윕** — 하네스로 누락0 유지하며 F1 상단 탐색.
4. **정확도 최우선이면 라지** — n2sf-xlmr-large 0.917(이미 0.8·0.85 초과).

> base 279M의 누락0 상한은 대략 0.80 부근으로 보임. 그 이상은 라지가 확실.

## 5.1 옵션1 실행 결과 — S·경계 증강 재증류(kd2)

교사(gpt-4o 생성 + o4-mini 라벨)로 **S 300 + 경계 100건 신규 증강** → S 366→647건(9%→15%). 증강셋으로 재증류(`models/n2sf-base-kd2`).

| 에폭 | 뉴럴-단독 F1 | C재현율 |
|---|---|---|
| **ep0** | **0.860** | 0.917 (누락 2) |
| ep1 | 0.824 | 0.958 (누락 1) |
| ep5 | 0.793 | 0.958 |

- **뉴럴-단독으론 0.86으로 0.8 훌쩍 초과**. 하지만 **이번엔 C재현율 1.0 에폭이 없음** — 증강한 S가 C/S 경계를 밀어 뉴럴 자체 기밀 재현율이 하락(S만 늘리고 **C 경계는 안 늘린 부작용**).
- **전체 파이프라인**(규칙 floor 포함) 재평가: **C재현율 1.00·유출 0** (규칙이 뉴럴 누락분을 전부 포착) — **운영상 안전**. 단 F1은 0.54로 규칙 **과대분류**(over_rate 0.39, 3차 앙상블 병목 재현) — 모델이 아니라 앙상블 튜닝 이슈.

**판정**
- "뉴럴-단독 0.8 + C재현율 1.0" 기준: **미달성**(최고 zero-leak 뉴럴은 kd의 0.796).
- "0.8 + 운영(전체 파이프라인) 누락 0" 기준: kd2 **충족**(뉴럴 0.86, 규칙 floor로 유출 0). 단 앙상블 과대분류는 별도 튜닝 필요.
- **kd2 미승격** — 뉴럴 C재현율이 base(1.0)보다 낮아(0.917) 방어 한 겹 약화. 현재 base(0.796, 뉴럴 C재현율 1.0) 유지.

**다음(진짜 0.8+zero-leak 뉴럴)**: S와 함께 **C 경계 문서도 증강**(양쪽 균형)해 C재현율을 지키며 F1↑, best-save에 **C재현율 1.0 하한** 강제.

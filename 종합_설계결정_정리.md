# 이노티움 등급분류 모델 — 종합 설계·결정 정리

> 일자: 2026-07-02 · 목적: 그동안의 실험·분석·설계 문서를 하나로 묶은 검토용 진입점
> 대상: `data_classifier.py`(정규식+NER+경량 BERT 3-tier, 외부 LLM·GPU 미사용, 온디바이스)

---

## 1. 한눈 요약 (지금까지)

| 단계 | 내용 | 결과 |
|---|---|---|
| 1차 | 합성셋 생성·자동 최적화·Gemma 비교 | 정상문서 LLM급, 초고속 |
| 2차 | 난이도 L1→L3, 적대 난독화 돌파 시도 | L3 "1.00"이나 **과적합** 판명 |
| 3차 | **soft-label 증류**(GPT-4o 교사→mdeberta-n2sf) | 학생 0.53→0.78, 파이프라인 **0.80**(누락 0) |
| 4차(진행) | 과대분류 억제(0.85 도전) | config 스윕 0.797 정체 → o4-mini 재증류 중 |
| 설계 | 모델 라인업·선택 구조·다국어 방어 | 본 정리 |

**현재 최고**: held-out(정직 분포) **macro-F1 0.80 / 기밀 재현율 1.00(누락 0)** — 온프레미스·CPU.
기준선: GPT-4o 0.915 · o4-mini 0.972 · Gemma 0.873.

---

## 2. 핵심 수치의 의미 (요약)

- "0.80" = **held-out(학습 미사용 72문서) macro-F1**. 실체 = **기밀 100% 포착(누락 0) + 하위등급 ~19% 과잉분류**.
- 0.80→0.85 = 과대분류 ~5건 축소(정밀도↑). **72문서라 ±0.03~0.05 노이즈** → 확정엔 평가셋 확대 필요.
- 상세: `수치의미_및_라지모델_검토.md` §1.

---

## 3. 현재 모델 & 크기 티어

- **mDeBERTa-v3-base** (BERT 계열 **base** 티어, 총 279M/558MB). "소형"은 **9B LLM 대비** 상대 표현.
- 상한 ~0.78~0.80 → **학생 용량이 병목**. 상세: `수치의미_및_라지모델_검토.md` §2.

---

## 4. LLM 지식증류 (요약)

- 교사(GPT-4o)의 **O/S/C 확률분포**를 학생(mDeBERTa)이 모방(soft-label KD). 실서비스 추론엔 **LLM 불필요**.
- 효과: 학생 0.53→0.78. **증류 상한 = 교사 성능**. 상세: `수치의미_및_라지모델_검토.md` §3.

---

## 5. 모델 라인업 & 선택 구조 (제안)

- **같은 교사로 소형/base/라지 3종 증류 → 용도별 선택.**
  - Fast(소형, KoELECTRA-small 등) / Balanced(base, 현재) / **Accurate(라지, KLUE-RoBERTa-large 등)**
- 선택: **정적 프로필 선택**(기본) 또는 **동적 캐스케이드**(쉬운 건 빠르게·애매한 건 강한 모델/LLM).
- 상세: `모델_라인업_선택구조_제안.md`.

---

## 6. 다국어 적대 공격 & 해소 (중요 리스크)

- **위협**: 한국어 특화(KLUE) 채택 시, 기밀을 **비한국어/혼용/로마자**로 써서 **누락(유출)** 유도 가능.
- **이미 방어**: T1 정규식·다국어 기밀 키워드는 **언어 무관** → 구조적 식별자·명시 라벨은 외국어라도 포착.
- **해소**: 다국어 백본 주력화 / 언어 라우팅 / Korean+다국어 앙상블(상향-only) / **다국어 적대 증류·평가(L5 레드팀)**.
- 상세: `다국어_적대공격_해소방안.md`.

---

## 7. 미해결·검증 필요

1. **실문서 검증** — 모든 수치는 합성 held-out 기준. 실서비스 전 실문서 표본 재검증 필수.
2. **0.85 달성 여부** — base 상한 ~0.80. 라지(KLUE-large/XLM-R-large) 증류로 0.83~0.88 기대(보장 아님).
3. **다국어 누락률** — 언어-적대 평가셋으로 상시 측정 필요.
4. **평가셋 규모** — 72→수백~수천으로 확대해 수치 신뢰도 확보.

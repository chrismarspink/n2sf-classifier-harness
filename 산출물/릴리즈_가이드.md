# N²SF 릴리즈 가이드 (item3) — 배포 방법

> 일자: 2026-07-03 · 목적: 학습된 모델·분류기를 **공개(HF Hub)** 또는 **서비스(Docker)** 로 배포하는 실행 절차.
> 원칙: 추론은 100% 로컬. 공개 배포보다 **통제된 배포**(사내/에어갭)가 우리 제품 기본.

---

## 1. 두 가지 배포 산출물

| 산출물 | 무엇 | 어디에 |
|---|---|---|
| **모델 가중치** | `models/n2sf-*` (HF 포맷) | Hugging Face Hub(공개/사설) · 사내 저장소 · tar 번들 |
| **분류 서비스** | 모델+엔진+API 컨테이너 | Docker Hub/사설 레지스트리 |

---

## 2. Hugging Face Hub 업로드 (가중치 공개/사설)

```bash
pip install huggingface_hub
huggingface-cli login                      # HF 토큰 입력(계정 필요)

# 공개 또는 사설(--private) repo로 업로드
huggingface-cli upload innotium/n2sf-xlmr-large ./models/n2sf-xlmr-large
huggingface-cli upload innotium/n2sf-base       ./models/n2sf-base
```

- **받기**: `AutoModelForSequenceClassification.from_pretrained("innotium/n2sf-base")`
- 엔진 연동: `data_classifier.py`의 `NEURAL_BACKENDS[...]["model"]` 값을 HF ID로 바꾸면 자동 다운로드.
- 공개범위: Public / Private(토큰) / Gated(약관 동의).

> ⚠️ 실행에는 HF 계정·토큰이 필요(현재 미보유). 토큰 주시면 실제 업로드까지 대행 가능.

---

## 3. Docker 이미지 (분류 서비스)

컨테이너에 **모델+엔진+HTTP API**를 담아 고객이 `docker run` 한 줄로 구동.

```bash
# 빌드(리포 루트에서)
docker build -t innotium/n2sf-classifier:1.0 -f 산출물/Dockerfile .

# 실행
docker run -p 8080:8080 innotium/n2sf-classifier:1.0

# 분류 요청
curl -s localhost:8080/classify -d '{"text":"대외비. 주민번호 900101-1234567"}'
#   → {"grade":"C","gradeLabel":"기밀","confidence":...,"tiersRun":["rules","ner"],"neuralSkipped":true,...}
curl -s localhost:8080/health
```

- 기본 모델 `n2sf-base`. 정확도 최우선이면 Dockerfile에서 `n2sf-xlmr-large` 복사 주석 해제 후
  `-e N2SF_MODEL=n2sf-xlmr-large`.
- **추론 100% 로컬 CPU** — 컨테이너는 외부 LLM/네트워크 호출 없음(에어갭 가능).
- 사설 레지스트리: `docker tag ... myregistry.local/n2sf-classifier:1.0 && docker push ...`

### 구성 파일
- `산출물/Dockerfile` — 이미지 정의
- `산출물/serve.py` — 표준 라이브러리 HTTP API(POST /classify, GET /health)
- `산출물/classify_n2sf.py` — 티어 early-exit 분류(튜닝 앙상블 기본 반영)

---

## 4. 에어갭·사내 배포 (공개 불가 고객)

- **tar 번들**: `tar czf n2sf-base.tgz models/n2sf-base` → USB/사내망 전달 → `from_pretrained("로컬경로")`.
- **사내 아티팩트**: Nexus·JFrog Artifactory·S3/MinIO에 모델 tgz 또는 Docker 이미지 등록.
- **오프라인 이미지**: `docker save innotium/n2sf-classifier:1.0 | gzip > n2sf.img.gz` → 고객 `docker load`.

---

## 5. 권장 실서비스 설정 (동봉)

- 앙상블: `soft`, 티어가중 `{rules:0.3, ner:0.3, neural:4.0}` (참조 소스 기본 반영).
- 근거·수치: `../앙상블_튜닝_결과.md`, `distill_soft/recommended_config.json`.
- 모델 선택: fast=n2sf-small / balanced=n2sf-base / accurate=n2sf-xlmr-large.

---

## 6. 버전·형상관리
- 이미지 태그 = 모델 버전과 일치(예: `n2sf-classifier:1.0` ↔ 모델 v1.0 스냅샷).
- HF repo도 태그/리비전 고정 권장(재현·감사).
- 상세 실체·라이선스: `../모델_실체_및_릴리즈_가이드.md`, 배포처 비교: `../모델_배포_레지스트리_안내.md`.

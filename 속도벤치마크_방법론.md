# 속도 벤치마크 — 방법론 & 예비 결과

> 목적: **속도(정확도 아님)** 측정. 용량 10~100MB × 등급(O/S/C) × 티어별 시간 + 어느 티어에서 확정 + 외부/로컬 LLM 대비.
> 요구: 100MB는 **모든 티어가 모든 텍스트를 누락없이** 검사.

---

## 1. 측정 방법 (누락 0 = 청킹 전수검사)

- 단일 대용량 문서를 그대로 넣으면 NER/뉴럴이 **truncate**(앞부분만) → 전체 검사 불가.
- 따라서 텍스트를 **4,000자 청크로 분할**, **모든 티어가 모든 청크를 검사**(뉴럴은 배치). → 누락 0.
- 측정 티어: **T1 규칙**(정규식+키워드) · **T2 NER**(Presidio+spaCy) · **T3 뉴럴**(배치 추론).
- **early-exit**: C(기밀표지)는 T1에서 확정 → T2/T3 불필요(실제 파이프라인 동작).

## 2. 공정 비교 (하드웨어 교란 제거) — 3자

| 대상 | 하드웨어 | 성격 |
|---|---|---|
| **우리 3-tier** | 맥미니 CPU | 온디바이스 제품 |
| **로컬 Gemma2:9B** | **동일 맥미니** | 온프레미스 LLM 대안(같은 HW=아키텍처 차이만) |
| **o4-mini** | 클라우드 GPU + 네트워크 | 클라우드 LLM 대안(과금·반출·rate limit) |

- 벽시계는 HW 의존 → **처리량(MB/s)** 병기. 우리 vs Gemma는 **동일 HW라 순수 비교**.
- LLM은 청크당 지연 실측 → 전수검사 시간 = 청크수×지연(직렬, 동시성/rate limit·비용 별도 주석).

## 3. 예비 실측 (1MB, 소형 뉴럴, CPU)

| 등급 | T1 규칙 | T2 NER | T3 뉴럴 | 전체 | **early-exit** | 확정 티어 |
|---|---|---|---|---|---|---|
| C(기밀표지) | 0.03s | 29.5s | 1.0s | 30.5s | **0.03s** | **T1(정규식)** |
| S(전체) | 0.03s | 29.4s | 1.0s | 30.4s | 30.4s | T3(뉴럴) |

**핵심 발견**
1. **병목 = NER(spaCy)** — ≈29s/MB. 뉴럴(≈1s/MB)·정규식(≈0.03s/MB)이 아니라 **NER이 속도 지배**.
2. **early-exit 위력** — C(기밀표지)는 T1에서 0.03s에 확정 → **전체 대비 ~1000×**. 명백한 기밀일수록 빠름.
3. → 실무 처리량은 **등급 믹스에 좌우**: C 비중이 높을수록 평균 훨씬 빠름.

## 4. 티어별 처리량(예비) & 100MB 외삽

| 티어 | 처리량(측정) | 100MB 전수(외삽) |
|---|---|---|
| T1 규칙 | ~32 MB/s | ~3s |
| **T2 NER** | **~0.034 MB/s** | **~49분** ← 병목 |
| T3 뉴럴(소형·배치) | ~1 MB/s | ~100s |

- 처리량은 용량에 **선형** → 10~100MB 표는 (처리량×용량)으로 정확 외삽 = 전수검사 시간과 동일.
- **100MB 전수검사 1회 실제 실행**으로 외삽 검증(누락0 요구 충족). → `final/speed_100mb.json`.

## 5. 산출(진행 중)
- `harness/speedbench.py` — 청킹 전수검사 벤치. `run_speedbench.sh`(진행 학습 후 자동 실행).
- `final/speed_throughput.json`(티어 처리량+Gemma/o4-mini) · `final/speed_100mb.json`(100MB 전수 검증).
- 완료 시 10~100MB × 등급 × (T1/T2/T3/전체/early-exit/vs Gemma/vs o4-mini) 상세 표 생성.

## 6. 시사점(예비)
- 대용량에서 **NER이 병목** → 실제 제품은 **early-exit(기밀표지·강식별자 즉시 확정)** + **필요 시 NER 스킵/샘플링**으로 가속.
- 우리 3-tier는 **소형 뉴럴 배치**로 뉴럴을 값싸게 처리 → LLM(청크당 수 초) 대비 큰 처리량 우위 예상(정량은 실행 후 확정).
- 정확도가 아닌 **속도·처리량·비용** 관점의 표 → 발표·용량 산정용.

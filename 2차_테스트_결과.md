# 2차 테스트 결과 — L3(적대 난독화) 돌파 시도

> 일자: 2026-06-30 · 목표: 1차에서 막힌 **L3(위장 문서, F1 0.54)** 돌파
> 신 모델명: **mdeberta-n2sf** (mDeBERTa-v3 파인튜닝) · 개선 레버: ①전처리 정규화 ②체크섬 ③파인튜닝
> 1차 자료는 `weekend/`·`harness_out/`에 보존. 2차는 `test2/`·`test2_phase1_normalize/`·`models/`.

## 1. 핵심 결과 (요약)

| 구성 | L3 macro-F1 | L3 기밀(C) 재현율 | 비고 |
|---|---|---|---|
| 1차(개선 없음) | **0.54** | 1.00 | 위장에 정규식 회피당함, 정체 |
| **2차 ① 정규화 적용** | **0.88** | 0.70 | 전각/공백 위장 무력화 → **대폭 상승**. 단, 난독 키워드 누락 |
| 2차 ③ mdeberta-n2sf 파인튜닝 | (학습 중, ETA ~2h) | 목표 ↑ | 난독 키워드 누락 회복 기대 |
| 비교: Gemma2:9b (L3) | (측정 중) | (측정 중) | 로컬 LLM의 L3 대응력 |

**한 줄**: **① 정규화만으로 L3 F1 0.54 → 0.88** (전각 주민번호·공백 위장이 정규식에 잡히게 됨). 남은 약점은 **난독 키워드(`대 외 비`)로 인한 기밀 누락(재현율 0.70)** — 이를 ③ 파인튜닝(mdeberta-n2sf)이 메우는지 검증 중.

---

## 2. 2차 테스트 구성 (무엇을 바꿨나)

세 가지 L3 돌파 레버를 하네스에 구현:

1. **① 전처리 정규화** (`harness/detect.py`) — 분류 전 텍스트를 **전각→반각** 변환, 공백 정리.
   `６８０１１９-...`(전각 주민번호) → `680119-...` → 정규식 적중. **즉효·결정론적·경량.**
2. **② 체크섬 검증** (계획) — 주민번호 검증자리·카드 Luhn 확인으로 가짜 식별자 오탐 제거(정밀도↑).
3. **③ mdeberta-n2sf 파인튜닝** (`harness/train.py`) — 제로샷 mDeBERTa에 **3-class(O/S/C) 헤드**를 얹어
   1차 데이터(L1~L3, 학습용 고유 21,600건)로 학습. 위장 패턴을 **예시로 일반화** → 난독 키워드 대응.

자동 평가: 하네스 `autoloop`를 **L3 고정**으로 돌려 모델·앙상블을 스윕하며 정규화·파인튜닝 효과를 측정.

---

## 3. 신 모델 구성 (현재 권장)

```
입력 문서
 → 추출 + ① 정규화(전각→반각·공백 정리)        ← 2차 신규
 → T1 정규식 / T2 NER / 키워드  → 점수·임계
 → T3 신경망: mdeberta(제로샷)  또는  mdeberta-n2sf(파인튜닝)   ← 2차 신규 백엔드
 → 앙상블: soft (현재 L3 최적)
 → 결정론적 강제상향(대량PII 등) → 등급
```

- **현재 L3 최적(파인튜닝 전)**: `① 정규화 ON + 뉴럴 mdeberta + soft 앙상블` → L3 F1 ≈ 0.88.
- **파인튜닝 후 목표**: `mdeberta-n2sf + soft` 로 기밀 재현율 0.70 → 0.95+ 회복(난독 키워드 잡기).
- mdeberta-n2sf 산출물: `models/mdeberta-n2sf/`(가중치 558MB, id2label=O/S/C). data_classifier `NEURAL_BACKENDS`에 `kind:"finetuned"`로 등록 완료.

---

## 4. 로컬 Gemma2:9b 대비 (L3)

> 1차(L1·정상)에서는 내 모델 1.00 vs Gemma 0.89, ~23배 빠름. 2차는 **L3(위장)** 기준 비교.

| 모델 | L3 macro-F1 | 기밀 재현율 | 비고 |
|---|---|---|---|
| 2차 3-tier (①정규화+mdeberta/soft) | **0.88** | 0.70 | CPU, ~수백ms |
| (목표) mdeberta-n2sf | 측정 중 | 측정 중 | 파인튜닝 후 |
| Gemma2:9b (로컬 LLM) | 측정 중 | 측정 중 | ~19s/건, GPU 자원 |

*(Gemma L3 수치는 측정 완료 시 갱신. 위장 문서에선 LLM 추론력이 유리할 수 있으나, 기밀문서 외부 반출 불가 제약은 1차 결론과 동일하게 유효.)*

## 5. 진행 중 / 이슈

- **mdeberta-n2sf 학습 진행 중** (MPS, batch16/maxlen128, 3에폭 ~1.5h). 150~300 step마다 체크포인트 저장.
- **이슈 기록**: 최초 학습이 **MPS 메모리 부족(OOM)**으로 1 step에서 실패 → 작은 배치 + 그래디언트 누적 + OOM 가드로 해결(재학습 정상).
- 완료 후 `test2/`에 자동 평가·리포트(report.html/WEEKEND_SUMMARY.md) 갱신.

## 6. 다음에 할 작업 (우선순위)

1. **mdeberta-n2sf 학습 완료 → L3 재평가** — 파인튜닝이 기밀 재현율(0.70)을 회복하는지 확정. (자동 진행 중)
2. **② 체크섬 검증 구현** — 주민번호/카드 진위 확인으로 가짜 식별자 오탐 제거(정밀도·과대분류 개선).
3. **③ 난독 키워드 정규화 보강** — `대 외 비` 같은 단일음절 공백까지 제거하는 키워드 전처리 추가(정규화로도 일부 회복 가능).
4. **Gemma L3 비교 완결** — 동일 L3 셋에서 3-tier vs Gemma 정확도/속도 표 확정.
5. **실문서 표본 확보 → 재검증·재보정** — 합성↔실전 격차 측정(상용화 전 필수).

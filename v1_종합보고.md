# N²SF 등급분류 — v1.0 종합 보고 & 다음 계획

> 일자: 2026-07-02 · 대상: 아침 보고용 종합 정리 · 버전: **v1.0**
> 요지: 외부 LLM·GPU 없이 온디바이스로 문서 보안등급(C/S/O) 분류. 증류로 **로컬 모델이 LLM급(0.917) 도달**,
> 전 모델 **기밀 누락 0**. 코드·문서·참조 소스·백서를 v1.0으로 확정. GitHub main 반영 완료.

---

## 0. 한 페이지 요약 (Executive Summary)

- **무엇을 만들었나**: 3-tier(정규식+NER+증류뉴럴) 온디바이스 등급분류기 + 다중 백본 라인업 + 참조 소스/백서.
- **핵심 성과**:
  - **XLM-R-large 증류 0.917 = 교사 GPT-4o(0.915) 동급** — "외부 LLM 없이 로컬에서 LLM급" 실측.
  - base(mDeBERTa) **0.719 → 0.796**(뉴럴 zero-leak) 개선·승격.
  - **전 모델·전 언어 기밀 누락 0**(한국어·영·중·일).
- **안전 원칙**: 과소분류(유출)=최악 → 기밀 재현율 최우선. held-out 정직 평가로만 보고.
- **v1.0 범위**: 코드·하네스·참조 소스(classify_n2sf.py)·백서·문서. 모델 가중치는 별도(HF/사내).
- **다음**: base 0.8+ zero-leak(C경계 균형증강), 앙상블 튜닝, GPT-5 교사, 다국어 교사라벨, 릴리즈(HF/Docker).

## 2. 최종 성능 (실측 · held-out 72문서 · 뉴럴-단독 · CPU)

### 2.1 라인업 3축 매트릭스
| 모델 | macroF1 | 기밀재현율 | 지연 p50 | 용량 | 파라미터 | 티어 |
|---|---|---|---|---|---|---|
| n2sf-small (KoELECTRA-small) | 0.709 | 1.00 | 13ms | 57MB | 14M | Fast |
| **n2sf-base (mDeBERTa, v1.1 GPT-5.4 증류)** | **0.888** | 1.00 | 122ms | 1.1GB | 279M | Balanced |
| n2sf-klue-large (KLUE-RoBERTa-large) | 0.872 | 1.00 | 205ms | 1.3GB | 337M | Korean |
| **n2sf-xlmr-large (XLM-R-large)** | **0.917** | 1.00 | 203ms | 2.3GB | 560M | Accurate |

> **v1.1 업데이트(2026-07-03)**: base를 GPT-5.4 교사로 균형 재증류 → 뉴럴-단독 0.796→**0.888**,
> **전체 파이프라인(튜닝 앙상블) 0.873**, C재현율 1.00. 앙상블 튜닝으로 전 모델 전체 파이프라인이
> 뉴럴 수준 회복(base 0.873 / xlmr 0.858, 모두 유출0). 상세: `GPT5_재증류_결과.md`, `앙상블_튜닝_결과.md`.

### 2.2 일반 LLM 대비
| 분류기 | macroF1 | 기밀재현율 | 반출 | 과금 |
|---|---|---|---|---|
| **n2sf-xlmr-large(로컬)** | **0.917** | 1.00 | 없음 | 0 |
| GPT-4o(교사·클라우드) | 0.915 | 1.00 | 있음 | 호출당 |
| o4-mini(클라우드) | 0.972 | 1.00 | 있음 | 호출당 |
| Gemma2:9b(로컬 LLM) | 0.873 | 0.917 | 없음 | 0 |

### 2.3 L5 다국어(영·중·일, 뉴럴-단독)
| 모델 | L5 macroF1 | 비한국어 기밀 누락 |
|---|---|---|
| n2sf-base(개선) | 0.730 | **0 / 0 / 0** |
| n2sf-klue-large | 0.805 | 0 / 0 / 0 |
| n2sf-xlmr-large | 0.556 | 0 / 0 / 0 |

> 발견: 다국어 XLM-R이 L5에서 낮음 — 교사라벨이 한국어 중심이라 다국어 용량 미발현(다음 계획 §6-4).

## 4. 빠른 실행 (재현·사용)

```bash
# 설치
pip install -r 산출물/requirements.txt
python -m spacy download ko_core_news_lg

# 분류(기본 base, 티어 early-exit)
python 산출물/classify_n2sf.py 문서.pdf
python 산출물/classify_n2sf.py 문서.docx --model n2sf-xlmr-large   # 정확도 최우선

# 평가/증류(하네스)
python -m harness.lineup            # 라인업 3축 재평가
python -m harness.eval_l5           # 다국어 누락률
python -m harness.train_kd --soft-jsonl distill_o4/teacher_labels.jsonl --balance …  # 재증류
```

## 5. 안전·정직 원칙 (불변)
- **기밀 재현율 최우선** — 과소분류(유출)=0 목표. 전 승격모델 뉴럴 C재현율 1.0.
- **정직 평가** — held-out(학습 미사용) 분포로만 성능 보고(과적합 착시 배제).
- **데이터 주권** — 교사엔 합성 문서만. 추론엔 LLM 호출 0. 에어갭 가능.
- ⚠️ **보안 조치 필요**: Azure 키가 아직 유효 → **폐기·재발급** 권장(작업 종료).

## 6. 다음 계획 (v1.1+ 로드맵)

### 6-1. base 0.8+ "뉴럴 zero-leak"으로 확실히 (우선)
- 문제: S만 증강 시 C/S 경계가 밀려 C재현율 하락(kd2 0.86이나 누락 2).
- 해법: **S와 C 경계 문서를 양쪽 균형 증강** + best-save에 **C재현율 1.0 하한** 강제.
- 목표: 뉴럴-단독 0.8+ & C재현율 1.0 동시 달성.

### 6-2. 앙상블 튜닝 (전체 파이프라인 과대분류 해소)
- 현상: 전체 파이프라인이 규칙 과대분류로 F1↓(0.54). 뉴럴 실력(0.8+)이 최종에 반영 안 됨.
- 해법: soft 앙상블 + 뉴럴 가중 상향(3차 0.797 방식) 재적용·하네스 스윕. 유출은 규칙 floor로 계속 0.

### 6-3. GPT-5 교사 재증류
- 키 확보 시 `.env.azure`에 배포 추가 → `distill.py --teacher gpt-5`. 라지 학생 + 강한 교사 → 상한↑.

### 6-4. 다국어 교사 라벨 (XLM-R 다국어 발현)
- 영/중/일 교사 라벨을 증류셋에 추가 → XLM-R의 다국어 용량 활성화 → L5 정확도↑.

### 6-5. 값싼 안전 강화
- 다국어 기밀표지(机密/極秘/CONFIDENTIAL)를 `GRADE_KEYWORDS`에 추가(규칙 tier 언어무관 검출).

### 6-6. 릴리즈·제품화
- **가중치 공개** → Hugging Face Hub / 사내 사설 repo.
- **서비스 배포** → Docker 이미지(모델+엔진+API) → 고객 `docker run` 한 줄.
- **정책 외부화** → GradeProfile(YAML)·floor_rules로 고객사별 등급체계·규정 주입.
- **고객 재보정** → 도입처 데이터로 임계·가중 재보정(하네스 재평가).

### 6-7. 검증 확대
- held-out 72→150+ 확대(측정 노이즈↓), 실문서 파일럿(합성→실데이터 갭 확인).

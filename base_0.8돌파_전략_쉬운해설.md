# 쉬운 해설서 — base 모델 점수 0.8 넘기기

> 일자: 2026-07-02 · 목적: 앞서 만든 `base_0.8돌파_전략.md`가 어렵다는 요청에 따라 **비유로 쉽게** 풀어 씀.
> 핵심 비유: **우리 모델 = 서류 분류를 배우는 "신입 직원"**, **LLM(GPT) = 가르치는 "선생님"**, **연습문제 = 학습 데이터**.

---

## 0. 큰 그림 한 문장

신입 직원(우리 모델)에게 서류를 **공개(O)/민감(S)/기밀(C)** 세 보관함에 나눠 넣는 법을 가르치는 중.
지금 시험 점수가 **72~78점(만점 100 기준, 0.72~0.78)**. **80점을 넘기고 싶다.**
문제는 신입 능력이 아니라 **가르치는 방식(연습 방법)**에 있다. 그걸 고치면 된다.

---

## 1. 지금 무엇이 문제인가 (쉬운 말로)

### 문제 ① "민감(S) 연습문제가 너무 적다" = **기아(굶주림)**
- 연습문제 3,874개 중 등급별 개수: **공개 1,326 / 민감 366 / 기밀 2,276**.
- 즉 **민감(S) 문제가 전체의 9%뿐.** 신입이 S를 거의 못 봐서 **S 판단을 못 배운다.**
- 그런데 실제 시험은 세 등급이 **똑같이 24문제씩** 나온다 → S를 못 배운 게 점수로 그대로 깎임.
- 👉 "기아 해소" = **민감(S) 연습을 굶지 않게 충분히 시키자**는 뜻.

### 문제 ② "선생님이 힌트를 안 준다" = 예/아니오만 알려줌
- 좋은 선생님은 "이건 **기밀 70%, 민감 30% 느낌**"처럼 **정도(뉘앙스)**를 알려준다.
- 그런데 지금 선생님(GPT)은 거의 다 **"이건 무조건 기밀!"** 처럼 딱 잘라 답한다(힌트 0%).
- 신입은 애매한 서류에서 "얼마나 민감한지"를 배울 기회가 없다 → 경계에서 실수.

### 문제 ③ "연습은 앞부분만, 시험은 더 길게" = 습관 불일치
- 연습할 땐 서류를 **앞 128글자만** 읽게 했는데, 시험 땐 **256글자**를 본다.
- 서류 뒷부분에 중요한 단서(주민번호 등)가 있으면 **연습 때 못 봐서** 놓친다.

---

## 2. 어떻게 고치나 (해결책을 쉬운 말로)

### 해결 ① 민감(S)을 충분히 연습시키기 (가장 중요)
세 가지 방법 중 골라서(또는 섞어서):

| 방법 | 쉬운 설명 |
|---|---|
| **균형 샘플링** | 연습시킬 때 공개·민감·기밀을 **같은 횟수로 번갈아** 보여준다. 적은 S를 **반복 출제**해 익히게 함. |
| **경계 문서 추가 생성** | "**경계 문서**" = 공개인지 민감인지, 민감인지 기밀인지 **애매한 서류**. 이런 헷갈리는 예제를 **더 만들어** 연습시킨다(선생님이 새로 출제). |
| **틀리면 벌점 크게 (가중 손실)** | 민감(S) 문제를 틀리면 **감점을 더 크게** 매겨서, 신입이 S에 더 신경 쓰게 만든다. |

> 효과: 신입이 S를 제대로 배우면 점수가 **가장 많이** 오른다(지금 S가 최대 약점이라서).

### 해결 ② 선생님이 "정도"까지 알려주게 만들기
- **온도(T) 조절**: 선생님의 딱 잘라 말한 답("무조건 기밀")을 **살짝 부드럽게 풀어서** "기밀 70·민감 30" 같은 **뉘앙스로 바꿔** 신입에게 전달. (코드 몇 줄이면 됨)
- 결과: 애매한 서류에서 "얼마나 민감한지 감"을 배워 **경계 실수가 준다.**

### 해결 ③ 연습·시험 조건 맞추기
- 연습도 **256글자**까지 읽게 통일 → 서류 뒷부분 단서도 학습.
- **공부 속도 조절**: 처음엔 천천히 시작(워밍업), 뒤로 갈수록 속도를 줄여(cosine) 안정적으로 마무리.
- **가장 좋았던 시점의 실력을 저장**(best 저장) — 연습 막판에 실력이 떨어져도 최고점을 남긴다.

### (여력 되면) 보조 방법
- **여러 시점 실력 평균내기(SWA)**: 연습 중 여러 순간의 실력을 **평균**하면 더 안정적. (공짜 이득)
- **시험문제 늘리기(72→150개+)**: 시험이 짧으면 운에 따라 점수가 흔들린다. 문제를 늘려 **진짜 실력**을 정확히 잰다.

## 3. 미리 알아둘 용어 (딱 이것만)

| 어려운 말 | 쉬운 뜻 |
|---|---|
| 클래스 | 등급(공개 O / 민감 S / 기밀 C) |
| 클래스 불균형 | 등급별 연습문제 개수가 크게 차이남 |
| 기아 해소 | 부족한 등급(S) 연습을 충분히 채워줌 |
| 균형 샘플링 | 등급을 같은 횟수로 번갈아 연습시킴 |
| 경계 문서 | 등급이 애매한(헷갈리는) 서류 |
| 가중 손실 | 특정 등급 틀리면 감점을 더 크게 |
| soft-label(소프트) | "70:30" 식 정도까지 알려주는 정답 |
| hard-label(하드) | "무조건 기밀" 식 딱 자른 정답 |
| 온도(T) | 딱 자른 정답을 부드러운 뉘앙스로 바꾸는 손잡이 |
| 증류 | 큰 선생님(LLM)의 판단을 작은 신입(우리 모델)에게 물려주기 |
| maxlen | 서류를 몇 글자까지 읽을지 |
| LR/스케줄 | 공부 속도와 그 조절 방식 |
| held-out | 연습에 안 쓴 새 시험문제(진짜 실력 측정용) |
| 기밀 재현율 | 진짜 기밀을 기밀로 맞추는 비율(놓치면 유출=최악) |

---

## 4. 그래서 뭘 하면 되나 (3줄 요약)

1. **민감(S) 연습을 많이 시킨다** (균형 샘플링 / 경계문서 추가 / S 감점 크게).
2. **선생님이 뉘앙스까지 알려주게 한다** (온도 T로 정답 부드럽게).
3. **연습·시험 조건을 맞춘다** (256글자, 공부속도 조절, 최고점 저장).

→ 이 셋을 반영해 **다시 한 번 학습**시키고, **새 시험문제로 점수 확인**. 80점 넘으면 성공.

**솔직한 기대**: 이 방식으로 **80~85점은 노려볼 만함**(보장은 아님). "확실히 더 높은 90점대"를 원하면 **더 큰 모델(XLM-R-large, 이미 92점)** 로 바꾸는 게 정답.
그리고 무슨 일이 있어도 **기밀을 놓치지 않는 것(재현율 100%)** 은 계속 지킨다.

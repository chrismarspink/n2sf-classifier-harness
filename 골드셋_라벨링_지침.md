# N²SF 등급 라벨링 지침 (골드셋 rubric)

> 일자: 2026-07-07 · 용도: 사람 라벨러가 문서를 C/S/O로 판정하는 절대 기준.
> **이 rubric은 재사용됨**: (1) 사람 골드셋 라벨링, (2) 제로샷 NLI 라벨 설명(`ZS_LABELS`),
> (3) exemplar 예시, (4) GradeProfile 규칙, (5) (옵션) 로컬 LLM 분류 프롬프트.

---

## 0. 대원칙

- **안전 우선**: 애매하면 **높은 등급**으로(과소분류=유출이 최악). "확신 없으면 한 단계 위."
- **문서 단위 판정**: 문서에 포함된 **가장 민감한 요소** 기준으로 등급 결정(가장 높은 것 하나면 그 등급).
- 라벨은 **내용 기준**(문체·형식 무관). 광고·공지여도 주민번호 있으면 C.

---

## 1. 등급 정의 & 결정 기준

### C · 기밀 (Confidential) — 유출 시 중대 피해
다음 중 **하나라도** 있으면 C:
- **강식별자**: 주민등록번호, 신용/체크카드번호, 여권번호, 계좌번호, API/암호키, 마이넘버.
- **명시적 기밀 라벨**: "기밀 / 대외비 / 극비 / 사외비 / CONFIDENTIAL / 機密 / 極秘 / 绝密".
- **대량 개인정보**(명부/내보내기): 개인 식별정보 10건 이상.
- 위가 **난독**(전각 숫자 `９０…`, 띄어쓴 `대 외 비`)돼 있어도 C.

### S · 민감 (Sensitive) — 제한적 개인정보·내부용
- **소수의 개인정보**(강식별자 아님): 이름+연락처/이메일, 단일 인물 정보.
- 내부 검토·업무 협의 등 **비공개 업무 문서**(강식별자·기밀표지 없음).
- **경계**: 개인정보가 1~2건이고 강식별자·기밀표지 없음 → S. 강식별자 나오면 C로 상향.

### O · 공개 (Open) — 유출돼도 무해
- **개인식별정보·기밀표지 없음**. 공지·안내·홍보·뉴스·공개 릴리스 등.
- 일반 명사로서의 "기밀/보안"(예: "정보보안 교육 안내")은 O일 수 있음 — **실제 기밀 내용이 없으면**.

---

## 2. 판정 순서 (결정 트리)

```
1) 강식별자(주민번호·카드·여권·계좌·키) 또는 명시적 기밀표지 있나?  → 예: C
2) 개인정보 10건 이상(명부)인가?                                  → 예: C
3) 소수 개인정보(이름+연락처 등) 또는 비공개 내부 업무 문서인가?    → 예: S
4) 개인정보·기밀 요소 전혀 없는 공개용 문서인가?                    → 예: O
5) 애매하면 → 한 단계 위 등급(안전 우선)
```

---

## 3. 경계 사례 가이드 (라벨러 혼동 방지)

| 상황 | 판정 | 이유 |
|---|---|---|
| "정보보안 지침 안내"(내용은 일반 공지) | O | 기밀 '내용'이 아니라 단어만 |
| "대외비"라고 적혀 있음(내용 무관) | C | 명시적 표지 = C floor |
| 이름+전화 1건 | S | 소수 개인정보 |
| 이름+전화+주민번호 | C | 강식별자 |
| 직원 명단 15명(이름+부서) | C | 대량 개인정보 |
| 계약서(금액·조건, 개인정보 없음) | S | 비공개 업무(강식별자 없음) |
| 전각 주민번호·띄어쓴 대외비 | C | 난독이어도 실체는 기밀 |

---

## 4. 라벨링 방법 (골드셋)

- 파일: `goldset_candidates.csv` (문서별 1행). 컬럼:
  `id, source, lang, format, model_pred, model_conf, text, human_label, note`
- **human_label** 칸에 `C/S/O` 입력. 애매하면 **note**에 사유 + 잠정등급.
- **2인 교차검수** 권장: 두 사람이 독립 라벨 → 불일치 건은 토론·확정(일치도 κ 기록).
- 목표: **500건**(등급·언어·생성기·난이도 층화). 경계/불확실 문서 우선 포함(학습·평가 가치 최대).

---

## 5. rubric의 다중 활용 (학습 외 기준 주입)

- **제로샷 라벨 설명**(`data_classifier.ZS_LABELS`): §1 정의를 자연어로 → 학습 없이 기준 반영.
- **exemplar**(`EXEMPLARS`): §3 경계 사례를 등급별 예시로 → 임베딩 기준 제시.
- **GradeProfile/floor_rules**: §1의 강식별자·표지·대량PII를 결정론적 규칙으로(이미 T1 반영).
- **(옵션) 로컬 LLM 프롬프트**: 본 지침 전문을 Gemma 등에 시스템 프롬프트로 → GPT식 rubric 분류.

---

## 6. 골드셋 용도

- **절대 기준**: LLM/정책 라벨이 아닌 사람 정답 → 교사·학생 정확도의 진짜 측정.
- **SLA 근거**: 기밀 누락률(C재현율) 보증 수치.
- **재보정**: 임계·규칙·프롬프트 튜닝의 정답셋.

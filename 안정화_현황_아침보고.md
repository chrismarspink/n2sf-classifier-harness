# 안정화 작업 현황 — 아침 보고용

> 일자: 2026-07-06 저녁 시작 · 대상: "점수 오락가락" 원인규명 + 근본 안정화.

---

## 1. 오락가락 원인 = 측정 노이즈 (A 결과, 확정)

부트스트랩 CI + 3시드 분산으로 정량화:

| 평가셋 | F1 | 95% CI | 폭 |
|---|---|---|---|
| 동일분포(72) | 0.838 | **[0.744, 0.917]** | ±0.085 (넓음) |
| OOD(417) | 0.706 | [0.667, 0.747] | ±0.04 |
| 시드 분산(3) | mean 0.656 | std **0.017** | 작음 |

- **주범 = 72문서 평가셋**(CI ±0.085). 0.958·0.888·0.838 등은 대부분 같은 구간 내 노이즈.
- **학습 시드 분산은 작음(0.017)** → 학습은 안정. 큰 평가셋+CI 쓰면 오락가락 소멸.

## 2. #1 다중 생성기 재증류 (완료)

- xlmr-robust: OOD(템플릿) **0.706→0.980**, 과대 0.245→0.022, 누출0.
- **단, 학습에 템플릿을 넣어 OOD가 "준-동일분포"화** → 순수 일반화 증명 아님. 동일분포(LLM)는 0.958→0.824로 분산.
- 해석: 분포 커버리지 확대(여러 스타일 고르게). 청정 검증 필요 → v2에서 수행.

## 3. 밤샘 안정화 v2 (진행 중, 아침 완료 예정)

`run_stabilize2.sh` — 진행:
1. **청정 미지분포 OOD**(o4-mini 생성 = 학습 미사용 4번째 스타일) → 진짜 일반화 수치.
2. **xlmr 다시드(1/2/3)** 강건셋 재학습 → 배포모델 분산 + **시드 앙상블**(분산 축소).
3. **3분포(동일/템플릿OOD/청정OOD) × (시드 CI + 앙상블)** 분석 → `final/stability_v2.json`.

**아침에 확인할 것**: `final/stability_v2.json`, `kd/stabilize2.log`
- 청정OOD F1 = 진짜 일반화(과적합 아닌 실력) 지표
- 시드 앙상블 F1/CI = 안정화된 배포 후보
- 예상 완료: xlmr 3시드(~6h) → 다음날 오전.

## 4. 지금까지 확정 결론 (정직)

- **대외 수치는 단일 점수 금지 → 구간+CI로.** 동일분포 0.84[0.74–0.92], 템플릿 0.71→(강건재증류)0.98(준-in-dist).
- **안전(누출0)은 라지+규칙floor로 견고**(#2 완료). 소형/base는 OOD 안전 약함.
- **오락가락은 평가셋 크기 문제** → 큰 held-out(≥400) 상시 사용 권장.
- 다음 근본책: **사람 골드셋 + 실문서**(자동 불가, 사람·데이터 필요).

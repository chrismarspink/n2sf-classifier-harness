# 맥 꺼짐/크래시 대비 — 작업 재개·복원 가이드

> 일자: 2026-07-03 · 질문: 맥북이 꺼지거나 VSCode가 죽으면 이어서 할 수 있나?
> 답 요약: **VSCode/터미널이 죽어도 학습은 계속됨(nohup 분리)**. **맥 전원이 꺼지면 학습만 멈추고, 저장된 체크포인트·커밋·대화는 살아있어 재개 가능.** 재실행 스크립트를 **재개형(완료분 건너뜀)** 으로 만들어 뒀음.

---

## 1. 상황별로 무슨 일이 일어나나

| 상황 | 학습(백그라운드) | 대화(Claude/VSCode) | 결론 |
|---|---|---|---|
| **VSCode 창만 닫음/죽음** (맥은 켜짐) | **계속 돌아감** (nohup+caffeinate로 분리됨) | 재접속하면 세션 복원 | 영향 거의 없음 |
| **맥 절전(sleep)** | caffeinate가 절전 막음(현재 sleep=0) | 유지 | 영향 없음 |
| **맥 전원 꺼짐/재부팅/배터리 방전** | **멈춤**(프로세스 종료) | 대화 transcript는 디스크에 있음 | **재개 필요**(아래 절차) |

> 즉 무서운 건 **전원 off** 하나뿐. 그 외(창 닫기·절전)는 안전.

---

## 2. 전원이 꺼져도 살아남는 것 (디스크·원격)

- ✅ **저장된 모델 체크포인트**: `train_kd`는 **에폭마다 best(held-out) 저장** → 진행 중이던 모델도 마지막 best는 디스크에 남음.
- ✅ **완료된 모델**: `models/n2sf-*-v2` 폴더 그대로.
- ✅ **교사 라벨셋**: `distill_o4/teacher_labels_gpt5.jsonl`.
- ✅ **깃 커밋**: GitHub에 푸시된 것(v1.0/v1.1)은 안전.
- ✅ **대화 기록**: Claude Code 세션 transcript(로컬 jsonl) → 세션 재개 가능.
- ✗ **잃는 것**: 실행 중이던 학습 프로세스, 아직 커밋 안 한 로컬 새 파일(디스크엔 있으나 백업 안 됨).

---

## 3. 재부팅 후 재개 절차 (3단계)

1. **VSCode 열기 → Claude Code 세션 재개**
   - VSCode 확장: 최근 세션 이어가기. (CLI라면 `claude --continue` 또는 `claude --resume`로 세션 선택.)
   - 나에게 "재부팅됨, 이어서" 하면 컨텍스트로 이어감.
2. **학습 재실행 = 재개형 스크립트**
   ```bash
   ./run_relineup.sh      # 이미 완료된 n2sf-*-v2 는 자동 건너뛰고 남은 것만 학습
   ```
   - `run_relineup.sh`는 **완료된 모델(config.json 존재)을 건너뛰도록** 수정됨 → 중단 지점부터 이어짐.
   - 반쯤 학습되다 죽은 모델: 그 폴더의 마지막 best로 남아 있으나, 확실히 하려면 그 폴더만 지우고 재실행(짧음).
3. **확인**
   ```bash
   tail -f kd/relineup.log            # 진행 로그
   ls -d models/n2sf-*-v2             # 완료 모델
   ```

---

## 4. 지금 적용된 안전장치

- **nohup + caffeinate**: 터미널/VSCode 분리 + 절전 방지(전원 있는 한 지속).
- **에폭별 best 저장**: 진행 손실 최소화(마지막 best는 항상 디스크).
- **재개형 오케스트레이션**: 완료분 건너뛰기 → 크래시해도 재실행 한 줄로 이어짐.
- **git 버전·태그**: 마일스톤(v1.0/v1.1)은 GitHub에 영구 보존.

---

## 5. 더 튼튼하게 하려면 (권장 설정)

- **전원 연결 유지**(배터리 방전 방지). 노트북이면 어댑터 필수.
- **전원복구 자동재시작**: 이미 `autorestart 1`(전원 복구 시 자동 부팅). `sudo pmset -a autorestart 1`로 확인/설정.
  - 단, 자동 부팅돼도 학습은 자동 재개 안 됨(로그인 후 `./run_relineup.sh` 필요). 완전 자동화하려면 launchd 등록 옵션 있음(원하면 구성).
- **주기적 커밋**: 중요한 산출물은 자주 커밋(로컬만 있으면 디스크 고장 시 유실).
- **오래 걸리는 작업은 Mac Mini에서**(주말 무인용, 전에 쓰던 방식) — 노트북보다 전원 안정.

---

## 6. 실사례 (방금 발생)

- 재증류 중 `train_kd`의 레이어 동결 코드 버그로 **klue/xlmr가 즉시 크래시**(small만 완료).
- 대응: 버그 수정 → **재개형 스크립트로 재실행** → **small은 건너뛰고 klue/xlmr만** 이어서 학습.
- 손실: small 재학습 0(건너뜀). 이게 바로 이 가이드의 재개 흐름입니다.

---

## 7. 한 줄 결론
> **창 닫기·절전은 안전(학습 지속). 전원만 조심.** 꺼져도 체크포인트·커밋·대화가 남아,
> `./run_relineup.sh`(완료분 자동 skip)로 **중단 지점부터 재개**된다.

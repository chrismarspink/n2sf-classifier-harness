# N²SF v3 — 완전 오프라인 배포 가이드 (HF 미접속 · NER 로컬)

> 원칙: **추론·NER·학습 전부 네트워크 접속 0.** 모델은 로컬 파일에서 직접 로드, spaCy(NER)도 설치본 로드.

---

## 1. 오프라인 보장 방식

| 구성요소 | 오프라인 방식 |
|---|---|
| **뉴럴 모델**(transformers) | `HF_HUB_OFFLINE=1`·`TRANSFORMERS_OFFLINE=1` 자동 설정(classifier_v3 최상단) → HF 접속 안 함. `from_pretrained(로컬디렉토리)`로만 로드 |
| **모델 파일** | `산출물_v3/models/<name>` 번들 우선 로드(없으면 리포 `models/`) |
| **NER**(Presidio+spaCy) | 설치된 spaCy 모델(`ko_core_news_sm`)을 로드만 — **다운로드 없음** |
| **문서 추출** | docx/pptx/hwpx=표준 zipfile, xlsx=openpyxl, pdf=pypdf — 전부 로컬 |

→ 실행 시 인터넷·HF·외부 LLM 접속 **일절 없음**.

---

## 2. 번들 만들기 (온라인 PC에서 1회)

```bash
cd 산출물_v3
./bundle_offline.sh                       # 권장 3티어(accurate/compact-multi/fast-ko) 복사
#   또는: ./bundle_offline.sh n2sf-xlmr-large n2sf-small
```
생성물:
- `산출물_v3/models/<name>/` — 모델 파일(로컬 로드용)
- `산출물_v3/offline_wheels/` — 파이썬 의존 + spaCy 모델 wheel

> spaCy 모델 wheel이 안 받아지면 온라인 PC에서:
> `python -m spacy download ko_core_news_sm && pip download -d offline_wheels ko_core_news_sm`

---

## 3. 폐쇄망 설치·실행

```bash
# (반입: 산출물_v3/ 통째로)
pip install --no-index --find-links offline_wheels -r requirements_offline.txt
pip install --no-index --find-links offline_wheels ko_core_news_sm    # NER 모델

# 실행 — 네트워크 없이 동작
python classifier_v3.py --text "[대외비] 내부 전략" --model accurate
python serve_v3.py        # 웹 서비스(:8080) + 학습/핫리로드
```

- classifier_v3 import 시 오프라인 환경변수가 자동 설정되므로 별도 조치 불필요.
- 강제 확인: `HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1 python classifier_v3.py --demo`

---

## 4. Docker 오프라인 이미지 (선택)

```dockerfile
FROM python:3.12-slim
ENV HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1
WORKDIR /app
COPY offline_wheels /wheels
COPY requirements_offline.txt .
RUN pip install --no-index --find-links /wheels -r requirements_offline.txt && \
    pip install --no-index --find-links /wheels ko_core_news_sm
COPY ../data_classifier.py /app/data_classifier.py
COPY . /app/산출물_v3
COPY models /app/models          # 또는 산출물_v3/models 사용
CMD ["python","산출물_v3/serve_v3.py"]
```
- 이미지에 모델·wheel·NER 모델 포함 → `docker run` 만으로 완전 오프라인 구동.
- 오프라인 이미지 반입: `docker save ... | gzip > n2sf.img.gz` → 폐쇄망 `docker load`.

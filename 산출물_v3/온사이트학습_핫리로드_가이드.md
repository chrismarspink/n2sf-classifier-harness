# N²SF v3 — 온사이트 재학습 & 무중단 핫리로드 가이드

> 목적: 실데이터 부족 문제를, **고객이 자기 문서에 라벨한 데이터로 현장에서 직접 재학습**해 해소.
> 학습→**무중단(zero-downtime) 모델 교체**까지 로컬(오프라인)에서.

---

## 1. 전체 흐름

```
① 고객 문서 라벨링(사람)      goldset_candidates.csv 형식 (text + human_label[C/S/O])
        │                     (라벨 기준 = N²SF §9: 개인정보=S, 국가안보·기밀표지=C, 공개=O)
        ▼
② 온사이트 학습               train_onsite.py — 기존 배포 모델에서 '이어서' 학습(전이) + 안정화 혼합
        │                     → models/n2sf-custom-vN/ (새 버전)
        ▼
③ 무중단 핫리로드             clf.reload_model('cust-vN', path='models/n2sf-custom-vN')
        │                     진행 중 요청=구모델 완료, 신규=신모델 → 다운타임 0
        ▼
④ 검증·롤백                   held-out 재평가. 나쁘면 reload_model(이전버전)으로 즉시 롤백.
```

---

## 2. ① 라벨 데이터 형식

CSV(UTF-8, 헤더) — `산출물/goldset_candidates.csv`와 동일 컬럼 사용 가능:
```
id,source,lang,chars,model_hint,uncertainty,text,human_label,note
G0001,,ko,120,,,"[대외비] 내부 전략 검토 …",C,
G0002,,ko,80,,,"채용 공고 안내 …",O,
```
- **필수**: `text`, `human_label`(C/S/O). 나머지는 선택.
- JSONL도 가능: `{"text":"...", "label":"C"}` (또는 grade/human_label).
- 라벨 기준서: `../골드셋_라벨링_지침.md` + `../N2SF공식기준_적용방안.md`(§9).

## 3. ② 온사이트 학습 (train_onsite.py)

```bash
python train_onsite.py \
  --labels customer_labels.csv \
  --base   models/n2sf-xlmr-large \        # 기존 배포 모델에서 이어 학습(전이)
  --out    models/n2sf-custom-v2 \
  --mix    distill_o4/teacher_labels_n2sf.jsonl --mix-ratio 0.5   # 안정화(망각 방지)
```
- **전이 학습**: `--base`를 기존 모델 디렉토리로 → 고객 소량 데이터로도 효과.
- **혼합(`--mix`)**: 기존 증류셋을 섞어 **파국적 망각** 방지(mix-ratio=고객:기존 균형).
- **라벨 스무딩**(기본 0.1): 소량·과신 방지.
- 함수 호출(웹/자동화):
  ```python
  from train_onsite import train_onsite
  res = train_onsite("customer_labels.csv", base="models/n2sf-xlmr-large",
                     out="models/n2sf-custom-v2", progress=lambda d: print(d))
  # res = {ok, model_dir, customer_labels, train_size, elapsed_s, next}
  ```
  `progress` 콜백으로 단계/에폭 로그 → **웹 UI 진행표시**에 연결.

## 4. ③ 무중단 핫리로드 (핵심)

```python
from classifier_v3 import N2SFClassifier
clf = N2SFClassifier("accurate")            # 서비스 기동
# ... 고객 학습 완료 후 ...
clf.reload_model("cust-v2", path="models/n2sf-custom-v2")
```
**동작 원리(다운타임 0)**:
1. 새 모델을 **먼저 로드·검증**(`_warm`) — 이 동안 기존 모델이 계속 서빙.
2. 활성 참조(`self._active`)를 **원자적으로 교체**(단일 참조 대입, GIL상 원자적).
3. 교체 시점 이전 요청은 **구 모델로 완료**, 이후 요청부터 신 모델.
4. `evict_old=True`면 구 모델 메모리 해제.

→ 재기동/연결 끊김 없음. 실패(로드 오류) 시 예외만 발생하고 **기존 모델 유지**(안전).

## 5. 웹 UI (serve_v3.py)

```bash
python serve_v3.py          # :8080
```
- `GET /` — 3패널 웹 UI: **분류 / 라벨파일 학습 / 핫리로드**.
- `POST /train {"labels_path":"..."}` — 백그라운드 학습(서비스 무중단) → `GET /train/status` 진행조회 →
  완료 시 **자동 핫리로드**(state=reloaded).
- `POST /reload {"model":"cust-v2","path":"models/n2sf-custom-v2"}` — 수동 교체.
- `POST /classify {"text":"..."}` — 분류.
- 학습이 백그라운드 스레드라 **학습 중에도 분류 서비스 계속**.

> 실 UI는 이 백엔드에 라벨링 화면(문서 표시→C/S/O 버튼→CSV 축적)만 붙이면 완성. 라벨링→학습→리로드 원클릭 파이프라인.

---

## 6. 운영·안전
- **버전 보존·롤백**: 각 학습은 새 디렉토리(vN). 성능 저하 시 `reload_model(이전vN)`로 즉시 롤백.
- **검증 게이트**: 리로드 전 held-out(또는 고객 검증셋) 자동 평가 → 기준 미달이면 리로드 보류(운영 정책).
- **오프라인**: 학습·리로드 전부 로컬. 외부 접속 0(오프라인 가이드 참조).
- **안전 원칙 유지**: §9 규칙 floor(기밀표지→C, 개인정보→S)는 재학습과 무관하게 항상 적용.

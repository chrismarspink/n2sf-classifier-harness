# N²SF 온프레미스 사용 방안

> 일자: 2026-07-03 · 질문: 허깅페이스로 어디서나 받게(도커 풀처럼) + 온프레미스 사용 방안 정리.
> 요지: **네, 맞습니다 — HF Hub은 "모델판 docker pull"**. `from_pretrained("org/model")` 한 줄로 어디서나 내려받아 실행.
> 온프레미스는 ① HF(공개/사설) 풀 ② Docker 이미지 ③ 에어갭 파일 번들 3가지로 커버.

---

## 1. "docker pull 처럼 모델도 풀 된다" — 맞음

| Docker | Hugging Face(모델) |
|---|---|
| `docker pull org/img` | `from_pretrained("org/model")` (자동 다운로드·캐시) |
| Docker Hub / 사설 레지스트리 | HF Hub / 사설 HF / 사내 저장소 |
| 이미지 태그 `:1.0` | 모델 리비전/태그 |

- 최초 1회 다운로드 후 로컬 캐시(`~/.cache/huggingface`) → 이후 오프라인 사용 가능.
- 업로드: `산출물/upload_hf.sh` (HF_TOKEN 필요). 받기: `from_pretrained("innotium/n2sf-base")`.

---

## 2. 온프레미스 3가지 방안 (환경별 선택)

### 2.1 인터넷 되는 사내망 — HF 사설(private) repo
```bash
export HF_TOKEN=hf_xxx
python -c "from transformers import AutoModelForSequenceClassification as M; M.from_pretrained('innotium/n2sf-base')"
```
- 사설 repo → 토큰 있는 조직원만 접근. 가장 간편한 배포/갱신.

### 2.2 서비스째 배포 — Docker 이미지 (권장, 통합 간단)
```bash
docker build -t innotium/n2sf-classifier:1.1 -f 산출물/Dockerfile .
docker run -p 8080:8080 innotium/n2sf-classifier:1.1
curl -s localhost:8080/classify -d '{"text":"대외비 주민번호 900101-1234567"}'
```
- **모델+엔진+API**가 이미지에 포함 → 고객은 `docker run` 한 줄. 인터넷 불필요(이미지 자체 포함).
- 사설 레지스트리: `docker tag ... myregistry.local/... && docker push`.

### 2.3 완전 에어갭(폐쇄망) — 파일 번들
```bash
# 반입: 모델 폴더 tar 또는 오프라인 도커 이미지
tar czf n2sf-base.tgz models/n2sf-base
docker save innotium/n2sf-classifier:1.1 | gzip > n2sf.img.gz
# 폐쇄망 내부
python -c "...from_pretrained('/opt/models/n2sf-base')"   # 로컬 경로 로드
docker load < n2sf.img.gz && docker run -p 8080:8080 innotium/n2sf-classifier:1.1
```
- 외부 접근 0. USB/승인 반입 매체로 전달. `from_pretrained`에 **로컬 경로** 지정하면 다운로드 없이 동작.

## 3. 하드웨어 요건 (No GPU)

| 티어 | 모델 | 메모리(대략) | 권장 |
|---|---|---|---|
| Fast(한국어) | n2sf-small | ~0.3GB RAM | 엣지·저사양 |
| Fast(다국어) | n2sf-small-multi-e5 | ~1GB RAM | 다국어 경량 |
| Balanced | n2sf-base | ~1.5GB RAM | 표준 업무 PC |
| Accurate | n2sf-xlmr-large | ~3GB RAM | 서버·정확도 최우선 |

- **GPU 불필요**. 일반 서버/PC CPU로 동작. 동시처리량은 코어 수에 비례(멀티프로세스/스레드).
- 더 작게(메모리↓): **int8 양자화·ONNX 변환**으로 2~4배 축소 가능(다음 단계, `주말 튜닝` 목표에 포함).

## 4. 운영 체크리스트

- [ ] 모델 배포처 결정(사설 HF / Docker / 번들) + 버전 태그 고정.
- [ ] 권장 앙상블 설정 적용(`recommended_config.json`: soft·규칙0.3·뉴럴4.0).
- [ ] 용도별 모델 선택(한국어전용→small, 다국어→small-multi/base, 정확도→xlmr).
- [ ] 고객 실문서로 **재보정**(임계·정책) + held-out 재평가(회귀 방지).
- [ ] GradeProfile로 회사 등급체계·규정 주입(선택).
- [ ] 추론 로그·감사(누락률 모니터링), 외부 네트워크 차단 확인(에어갭).

## 5. 결론
- **"어디서나 받아 쓰기" = HF Hub(공개/사설), docker pull과 동일 개념.**
- 온프레미스는 **HF 사설 / Docker 이미지 / 에어갭 번들** 중 환경에 맞게 선택. 전부 **로컬 CPU·외부호출 0**.
- 최소 메모리는 소형 모델 + (향후) 양자화로 달성. 정확도-크기 trade-off는 최종 매트릭스(`final/report.md`)로 선택.

# 앙상블 튜닝 결과 (item2) — 전체 파이프라인 과대분류 해소

> 일자: 2026-07-03 · 목적: 전체 3-tier 파이프라인의 과대분류(F1 0.54)를 잡아 **뉴럴 실력을 최종 등급에 반영**.
> 결과: 앙상블 가중만 조정해 **전체 파이프라인 F1 0.54 → 0.80(base)·0.86(xlmr)**, **C재현율 1.0(누락0) 유지**.

---

## 1. 문제

- 뉴럴-단독은 좋은데(base 0.796, xlmr 0.917), **전체 파이프라인은 F1 0.54**로 붕괴(과대분류 over_rate 0.39).
- 원인: 기본 앙상블이 규칙/NER을 뉴럴과 **동일 가중(1.0)** 으로 섞어, 규칙의 과대분류가 최종 등급을 밀어올림.

## 2. 해법 — 뉴럴 중심 soft 앙상블

`soft` 앙상블에서 **규칙/NER 가중은 낮추고(floor 역할만) 뉴럴 가중을 높임**.
강식별자·대량PII는 별도 **컴플라이언스 floor**(하향 불가)가 지키므로 안전은 유지.

## 3. 스윕 결과 (n2sf-base, held-out 72, 전체 파이프라인)

| 앙상블 | 규칙가중 | 뉴럴가중 | F1 | C재현율 | 과대분류 | 유출 |
|---|---|---|---|---|---|---|
| escalate(기본) | 1.0 | 1.0 | 0.514 | 1.00 | 0.40 | 0 |
| soft(기본) | 1.0 | 1.0 | 0.540 | 1.00 | 0.39 | 0 |
| soft | 0.5 | 2.0 | 0.744 | 1.00 | 0.24 | 0 |
| soft | 0.3 | 3.0 | 0.762 | 1.00 | 0.22 | 0 |
| **soft** | **0.3** | **4.0** | **0.796** | **1.00** | 0.19 | 0 |

→ **뉴럴 실력(0.796)을 전체 파이프라인에서 그대로 회복**, 누락 0 유지.

## 4. 전 모델 적용 (soft · 규칙0.3 · 뉴럴4.0)

| 모델 | 전체 파이프라인 F1 | C재현율 | 유출 |
|---|---|---|---|
| n2sf-base | 0.796 | 1.00 | 0 |
| n2sf-klue-large | 0.826 | 1.00 | 0 |
| **n2sf-xlmr-large** | **0.858** | 1.00 | 0 |

- 라지도 회복(xlmr 0.86). 뉴럴-단독(0.917)엔 약간 못 미침 — 규칙 floor의 잔여 과대분류(over 0.14). 뉴럴가중 더↑로 근접 가능.

## 5. 권장 실서비스 설정 (확정)

```json
{ "ensemble_method": "soft", "tier_weights": {"rules": 0.3, "ner": 0.3, "neural": 4.0} }
```
- `distill_soft/recommended_config.json`에 저장. 참조 소스 `classify_n2sf.py`의 `TUNED_TIER_WEIGHTS`에 기본 반영.
- 안전: 강식별자·대량PII·마이넘버는 컴플라이언스 floor가 별도 보장(앙상블 가중과 무관).

## 6. 결론
- 전체 파이프라인 과대분류는 **모델이 아니라 앙상블 가중 문제**였고, **재학습 없이** 해소.
- 실서비스 기대치: base **0.80** / xlmr **0.86**, **누락 0**. (뉴럴-단독 상한에 근접)
